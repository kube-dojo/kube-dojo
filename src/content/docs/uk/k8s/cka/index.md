> **Certified Kubernetes Administrator** — Продемонструйте свою здатність встановлювати, налаштовувати та керувати кластерами Kubernetes промислового рівня

## Про CKA

CKA — це **практичний іспит на основі продуктивності**, який перевіряє ваші навички адміністрування Kubernetes та управління кластерами.

| Аспект | Деталі |
|--------|--------|
| **Формат** | На основі продуктивності (практичний) |
| **Тривалість** | 2 години |
| **Запитання** | 15-20 завдань |
| **Прохідний бал** | 66% |
| **Термін дії** | 3 роки |

## Структура навчальної програми

| Частина | Тема | Вага | Модулі |
|---------|------|------|--------|
| Частина 0 | Середовище | - | 6 |
| Частина 1 | Архітектура кластера, встановлення та налаштування | 25% | 8 |
| Частина 2 | Робочі навантаження та планування | 15% | 8 |
| Частина 3 | Сервіси та мережа | 20% | 8 |
| Частина 4 | Сховище | 10% | 6 |
| Частина 5 | Усунення несправностей | 30% | 8 |
| **Усього** | | **100%** | **44** |

## Огляд модулів

### Частина 0: Середовище (6 модулів)
- 0.1 Налаштування кластера — Створення практичних кластерів
- 0.2 Майстерність оболонки — bash, псевдоніми, автодоповнення
- 0.3 Vim та YAML — Ефективне редагування
- 0.4 Документація Kubernetes — Навігація документацією
- 0.5 Стратегія іспиту — Підхід трьох проходів
- Підсумковий тест частини 0

### Частина 1: Архітектура кластера (8 модулів)
- 1.1 Площина управління — API server, etcd, scheduler
- 1.2 Інтерфейси розширення — CNI, CSI, CRI
- 1.3 Helm — Управління пакетами
- 1.4 Kustomize — Управління конфігурацією
- 1.5 CRDs та Operators — Розширення Kubernetes
- 1.6 RBAC — Контроль доступу
- 1.7 kubeadm — Розгортання та оновлення кластера
- Підсумковий тест частини 1

### Частина 2: Робочі навантаження та планування (8 модулів)
- 2.1 Поди — Базова одиниця робочого навантаження
- 2.2 Деплойменти — Управління застосунками
- 2.3 DaemonSets та StatefulSets — Спеціалізовані навантаження
- 2.4 Jobs та CronJobs — Пакетна обробка
- 2.5 Управління ресурсами — Запити, ліміти, квоти
- 2.6 Планування — Вибір вузлів, спорідненість
- 2.7 ConfigMaps та Secrets — Конфігурація
- 2.8 Планувальник та теорія життєвого циклу подів
- 2.9 Автомасштабування — HPA, VPA, metrics-server
- Підсумковий тест частини 2

### Частина 3: Сервіси та мережа (8 модулів)
- 3.1 Сервіси — ClusterIP, NodePort, LoadBalancer
- 3.2 Endpoints — Бекенди сервісів
- 3.3 DNS — CoreDNS та виявлення сервісів
- 3.4 Ingress — HTTP-маршрутизація
- 3.5 Gateway API — Наступне покоління ingress
- 3.6 Network Policies — Контроль трафіку
- 3.7 CNI — Мережа контейнерів
- Підсумковий тест частини 3

### Частина 4: Сховище (6 модулів)
- 4.1 Volumes — Основи сховища
- 4.2 PV та PVC — Постійне сховище
- 4.3 StorageClasses — Динамічне виділення
- 4.4 Volume Snapshots — Резервне копіювання та відновлення
- 4.5 Усунення несправностей сховища — Типові проблеми
- Підсумковий тест частини 4

### Частина 5: Усунення несправностей (8 модулів)
- 5.1 Методологія — Систематичний підхід
- 5.2 Збої застосунків — Проблеми подів та контейнерів
- 5.3 Площина управління — Проблеми API server, etcd
- 5.4 Робочі вузли — kubelet, мережа
- 5.5 Мережа — DNS, з'єднання
- 5.6 Сервіси — Проблеми балансування навантаження
- 5.7 Логування та моніторинг — Спостережуваність
- Підсумковий тест частини 5

## Як користуватися цією програмою

1. **Налаштуйте середовище** — Спочатку пройдіть частину 0
2. **Опануйте основи** — Частина 1 є фундаментальною
3. **Практикуйте усунення несправностей** — Це 30% іспиту
4. **Використовуйте стратегію трьох проходів** — Спочатку швидкі перемоги
5. **Засікайте час** — Набирайте швидкість для іспиту

## Ключові навички

- Встановлення кластера за допомогою kubeadm
- Налаштування RBAC
- Network policies
- Постійне сховище
- Методологія усунення несправностей

## Почніть навчання

Починайте з частини 0, щоб налаштувати середовище та вивчити стратегії іспиту.

Успіхів на шляху до CKA!
