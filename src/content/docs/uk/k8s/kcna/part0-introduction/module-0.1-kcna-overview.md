## Чому цей модуль важливий

KCNA (Kubernetes and Cloud Native Associate) — це **точка входу** до сертифікацій Kubernetes. На відміну від CKA або CKAD, це іспит з множинним вибором, який перевіряє ваше концептуальне розуміння — а не вашу здатність вводити команди kubectl під тиском.

Це робить його ідеальним для:
- Менеджерів, яким потрібно розуміти Kubernetes
- Розробників, нових у хмарних технологіях
- Усіх, хто починає свій шлях у Kubernetes
- ІТ-фахівців, які переходять до хмарних технологій

## Підхід до навчання

Оскільки KCNA — концептуальний, ваш підхід до навчання відрізняється від практичних іспитів:

### Робіть це

1. **Розумійте "чому"** - Знайте, чому існує кожна концепція
2. **Вивчайте термінологію** - Знайте, що означають терміни
3. **Вивчайте діаграми** - Візуалізуйте архітектуру
4. **Проходьте практичні вікторини** - Практика множинного вибору
5. **Досліджуйте ландшафт CNCF** - Знайте основні проєкти

### Не робіть цього

- Не заучуйте команди kubectl
- Не витрачайте години на синтаксис YAML
- Не будуйте складні кластери
- Не хвилюйтеся через граничні випадки

## Приклади питань

Ось як виглядають питання KCNA:

### Питання 1
**Яка найменша одиниця розгортання в Kubernetes?**
- A) Container
- B) Pod
- C) Deployment
- D) Node

<details>
<summary>Відповідь</summary>
B) Pod. Хоча контейнери працюють всередині Подів, Под — це найменша одиниця, якою керує Kubernetes.
</details>

### Питання 2
**Який компонент відповідає за планування Подів на вузли?**
- A) kubelet
- B) kube-proxy
- C) kube-scheduler
- D) etcd

<details>
<summary>Відповідь</summary>
C) kube-scheduler. Він спостерігає за щойно створеними Подами та призначає їх на вузли.
</details>

### Питання 3
**Що означає CNCF?**
- A) Cloud Native Computing Foundation
- B) Container Network Configuration Framework
- C) Cloud Networking and Container Federation
- D) Container Native Cloud Foundation

<details>
<summary>Відповідь</summary>
A) Cloud Native Computing Foundation. CNCF розміщує Kubernetes та інші хмарні проєкти.
</details>

## Чи знали ви?

- **KCNA запущено у 2021 році** як першу сертифікацію початкового рівня для Kubernetes. До цього CKA був єдиним варіантом.

- **Вимога 75% для проходження** означає, що ви можете пропустити близько 15 питань і все одно скласти. Це м'якше, ніж 66% у CKA.

- **Без практичної частини означає без kubectl** - Ви не введете жодної команди під час іспиту. Тільки читання та вибір відповідей.

- **Іспит змінюється** - Оновлення навчальної програми відбулося у листопаді 2025. Слідкуйте за оголошеннями CNCF.

## Підсумок

**KCNA — це ваша точка входу** до сертифікації Kubernetes:

- **Формат**: 90 хвилин, ~60 множинний вибір, 75% для проходження
- **Фокус**: Концепції, а не команди
- **Найбільший домен**: Основи Kubernetes (46%)
- **Підхід до навчання**: Розумійте "чому", вивчайте термінологію, досліджуйте екосистему

**Це відрізняється від CKA/CKAD/CKS**:
- Без доступу до терміналу
- Без команд kubectl
- Без написання YAML
- Чисте концептуальне розуміння
