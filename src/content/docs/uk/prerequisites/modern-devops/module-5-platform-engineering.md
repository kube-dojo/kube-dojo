## Чому цей модуль важливий

Коли організації впроваджують Kubernetes, виникає проблема: Kubernetes складний, і не кожен розробник хоче (або повинен) глибоко його розуміти. Платформна інженерія вирішує цю проблему, створюючи внутрішні платформи, що абстрагують складність, дозволяючи розробникам зосередитися на коді, поки платформні команди займаються інфраструктурою. Розуміння цього тренду допоможе вам ефективно позиціонувати свої навички.

### 2. Золоті шляхи

Попередньо побудовані, рекомендовані способи виконання типових завдань:

```yaml
# Замість: "Розберися, як деплоїти в Kubernetes"
# Золотий шлях: "Використай цей шаблон"

# scaffold/microservice-template/template.yaml
apiVersion: scaffolder.backstage.io/v1beta3
kind: Template
metadata:
  name: microservice-template
  title: Create a new microservice
spec:
  parameters:
    - name: serviceName
      description: Name of your service
    - name: team
      description: Your team name
    - name: language
      options: [go, python, java, node]

  steps:
    - id: create-repo
      action: github:create-repo
    - id: add-ci-cd
      action: add-github-actions
    - id: register-service
      action: backstage:register
```

## Crossplane: Абстракція інфраструктури

Crossplane дозволяє визначати інфраструктуру як ресурси Kubernetes:

```yaml
# Платформна команда визначає CompositeResourceDefinition
apiVersion: apiextensions.crossplane.io/v1
kind: CompositeResourceDefinition
metadata:
  name: databases.platform.company.io
spec:
  group: platform.company.io
  names:
    kind: Database
  versions:
    - name: v1
      schema:
        openAPIV3Schema:
          type: object
          properties:
            spec:
              type: object
              properties:
                size:
                  type: string
                  enum: [small, medium, large]

---
# Розробник просто запитує:
apiVersion: platform.company.io/v1
kind: Database
metadata:
  name: my-db
spec:
  size: small

# Платформа обробляє складні ресурси AWS/Azure/GCP
```

## Чи знали ви?

- **Backstage створили у Spotify** для керування 1 500+ мікросервісами. Його відкрили у 2020 році, і тепер це CNCF Incubating проєкт, який використовують тисячі компаній.

- **Платформна інженерія — тренд №1** серед стратегічних технологічних трендів Gartner 2024 року. Вони прогнозують, що 80% організацій з розробки програмного забезпечення матимуть платформні команди до 2026 року.

- **Платформа Netflix** дозволяє розробникам деплоїти сотні разів на день серед тисяч мікросервісів з мінімальним залученням платформної команди.

## Типові помилки

| Помилка | Чому це шкодить | Рішення |
|---------|-----------------|---------|
| Створення без зворотного зв'язку | Платформа, яка нікому не потрібна | Спочатку поговоріть з розробниками |
| Занадто багато абстракції | Розробники не можуть дебажити | Надайте можливість «зазирнути під капот» |
| Відсутність золотих шляхів | Кожна команда винаходить свій підхід | Визначте та просувайте найкращі практики |
| Платформа як вартовий | Уповільнює всіх | Платформа має сприяти, а не блокувати |
| "Побудуй — і вони прийдуть" | Низьке впровадження | Ставтеся до платформи як до продукту, просувайте її |

## Тест

1. **Яку проблему вирішує платформна інженерія?**
   <details>
   <summary>Відповідь</summary>
   Когнітивне перевантаження. Розробники не повинні розуміти весь Kubernetes, CI/CD, моніторинг тощо. Платформна інженерія надає абстракції самообслуговування, що приховують складність, зберігаючи при цьому стандарти.
   </details>

2. **Що таке «золотий шлях»?**
   <details>
   <summary>Відповідь</summary>
   Попередньо побудований, рекомендований спосіб виконання типових завдань. Замість того, щоб розбиратися, як деплоїти в Kubernetes, розробники використовують шаблон, який автоматично забезпечує найкращі практики.
   </details>

3. **Чим платформна інженерія відрізняється від DevOps?**
   <details>
   <summary>Відповідь</summary>
   DevOps говорив: «Ти це створив — ти й керуєш» (кожна команда робить все сама). Платформна інженерія створює виділену команду, яка будує повторно використовувану інфраструктуру, щоб продуктові команди могли зосередитися на бізнес-цінності.
   </details>

4. **Що таке Backstage?**
   <details>
   <summary>Відповідь</summary>
   Відкритий портал для розробників від Spotify (тепер у CNCF). Він надає каталог сервісів, шаблони для створення нових сервісів та систему документації. Це де-факто стандарт для внутрішніх платформ розробників.
   </details>

kind: Deployment
metadata:
  name: myapp
  labels:
    app: myapp
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: nginx:1.25
        ports:
        - containerPort: 80

          requests:
            memory: "64Mi"
            cpu: "100m"

            memory: "128Mi"
            cpu: "200m"
        livenessProbe:
          httpGet:
            path: /
            port: 80
        readinessProbe:
          httpGet:
            path: /
            port: 80

kind: Service

  name: myapp

  selector:
    app: myapp
  ports:
  - port: 80
EOF

kubectl apply -f complex-deployment.yaml

# 2. "Платформний спосіб" (спрощений інтерфейс)
# Уявіть платформу, де розробник просто каже:
cat << 'EOF'
# platform-request.yaml (гіпотетичний)
name: myapp

kubectl get deployment myapp

# - Обмеження (ліміти, безпека тощо)

kubectl delete -f complex-deployment.yaml
rm complex-deployment.yaml

## Підсумок

**Платформна інженерія** — це створення кращого досвіду для розробників:

**Основна ідея**:
- Внутрішня платформа для розробників (IDP)
- Самообслуговування з обмеженнями
- «Платформа як продукт»

**Ключові компоненти**:
- Портал розробника (Backstage)
- Золоті шляхи (шаблони)
- Інфраструктура самообслуговування

**Інструменти**:
- Backstage для порталів
- Crossplane для інфраструктури
- Kubernetes як основа

**Чому це важливо**:
- Зменшує когнітивне навантаження
- Підвищує продуктивність розробників
- Підтримує стандарти та безпеку
- Дозволяє масштабувати практики DevOps

**Для вас**: Розуміння платформної інженерії допоможе вам створювати кращі платформи АБО бути ефективнішим користувачем платформи.
