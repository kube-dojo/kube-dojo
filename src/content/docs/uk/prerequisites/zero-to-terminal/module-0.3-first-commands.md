## Чому цей модуль важливий

Термінал — це те, як професіонали спілкуються з комп'ютерами. Клацання кнопок у графічному інтерфейсі підходить для повсякденних завдань, але коли потрібно керувати серверами, автоматизувати роботу чи використовувати Kubernetes — термінал є вашим основним інструментом.

Ось у чому справа: **термінал не складніший за графічний інтерфейс. Він просто інший.** Замість клацання на теці, щоб її відкрити, ви вводите команду. Замість перетягування файлу для переміщення — вводите команду. Ті самі дії, інший метод.

До кінця цього модуля ви знатимете 9 команд, які покривають приблизно 80% повсякденної роботи в терміналі.

---

## Відкриваємо термінал

Перед початком вам потрібно відкрити термінал.

**macOS**: Натисніть `Cmd + Space`, введіть «Terminal», натисніть Enter. (Або знайдіть його в Applications > Utilities > Terminal.)

**Windows**: Знайдіть «PowerShell» у меню Start. (Більшість наведених нижче команд працюють у PowerShell. Для повної сумісності встановіть [Windows Terminal](https://aka.ms/terminal) та [WSL](https://learn.microsoft.com/en-us/windows/wsl/install) — але не переймайтесь цим сьогодні.)

**Linux**: Натисніть `Ctrl + Alt + T` на більшості систем або знайдіть «Terminal» у ваших додатках.

Ви маєте побачити щось на кшталт:

```
username@computername ~ $
```

Цей `$` (або `%` на деяких Mac) — це **рядок запрошення**. Він означає, що термінал чекає, поки ви щось введете. Уявіть, що кухонний персонал каже: «Замовлення, будь ласка!»

## Ваша файлова система: План ресторану

Перед тим як почати виконувати команди, потрібно зрозуміти одну річ: ваш комп'ютер організовує файли у **деревоподібну структуру**. Уявіть це як кімнати в будівлі.

```
/ (корінь -- сама будівля)
├── Users/
│   └── yourname/          ← Це ваша "домашня директорія"
│       ├── Desktop/
│       ├── Documents/
│       ├── Downloads/
│       └── Pictures/
├── Applications/
└── System/
```

Кожен файл живе десь у цьому дереві. Команди дають змогу переміщуватися деревом, бачити вміст кожної кімнати та створювати чи видаляти речі.

## Команда 1: `pwd` — «Де я?»

**pwd** означає **P**rint **W**orking **D**irectory (вивести робочу директорію). Вона показує, де ви зараз.

Уявіть, що ви питаєте: «В якій кімнаті я перебуваю?»

```bash
pwd
```

Очікуваний результат:
```
/Users/yourname
```

(На Linux це може бути `/home/yourname`. У Windows PowerShell — щось на кшталт `C:\Users\yourname`.)

Це ваша **домашня директорія** — ваш особистий простір на комп'ютері. Це як ваш власний кабінет у ресторані.

**Коли використовувати**: Коли не впевнені, де ви. Навіть досвідчені інженери постійно вводять `pwd`. Немає нічого соромного в перевірці.

---

## Команда 2: `ls` — «Що тут є?»

**ls** означає **L**i**s**t (список). Вона показує, що є у вашій поточній директорії (кімнаті).

```bash
ls
```

Очікуваний результат (у вас буде інакше):
```
Desktop    Documents    Downloads    Music    Pictures
```

Хочете більше деталей? Додайте прапорець `-l` (маленька L, від «long format» — довгий формат):

```bash
ls -l
```

Очікуваний результат:
```
drwx------   4 yourname  staff   128 Mar 15 10:30 Desktop
drwx------   5 yourname  staff   160 Mar 20 09:15 Documents
drwx------  12 yourname  staff   384 Mar 22 14:45 Downloads
```

Не переймайтесь, якщо не розумієте кожен стовпець. Важливі частини — це назва (праворуч) і дата (коли останній раз змінювали).

Хочете побачити й приховані файли? (Файли, що починаються з крапки, як `.bashrc`, приховані за замовчуванням.)

```bash
ls -la
```

Прапорець `-a` означає «all» (усе) — показати все, включно з прихованими файлами.

## Команда 3: `cd` — «Перейти кудись»

**cd** означає **C**hange **D**irectory (змінити директорію). Вона переміщує вас в іншу кімнату.

```bash
cd Documents
```

Тепер перевірте, де ви:

```bash
pwd
```

Результат:
```
/Users/yourname/Documents
```

Ви перемістились! Тепер ви «всередині» теки Documents.

### Повернутися назад: `cd ..`

`..` означає «батьківська директорія» — кімната, яка містить цю кімнату.

```bash
cd ..
```

```bash
pwd
```

Результат:
```
/Users/yourname
```

Ви повернулись у домашню директорію.

### Повернутися додому: `cd ~`

Де б ви не були у файловій системі, `cd ~` повертає вас додому. Символ `~` (називається «тильда») — це скорочення для вашої домашньої директорії.

```bash
cd ~
```

Це як кнопка «повернутися на базу». Використовуйте її, коли заблукали.

### Перейти в конкретне місце: `cd /path/to/place`

Ви можете перейти безпосередньо в будь-яке місце, ввівши повний шлях:

```bash
cd /tmp
```

```bash
pwd
```

Результат:
```
/tmp
```

Тепер поверніться додому:

```bash
cd ~
```

## Команда 4: `mkdir` — «Побудувати нову кімнату»

**mkdir** означає **M**a**k**e **Dir**ectory (створити директорію). Вона створює нову теку.

```bash
mkdir my-first-folder
```

Перевірте, що спрацювало:

```bash
ls
```

Ви маєте побачити `my-first-folder` у списку.

### Створення вкладених тек: `mkdir -p`

Що, якщо ви хочете створити теку всередині теки всередині теки? Прапорець `-p` (від «parents» — батьківські) створює весь шлях одразу:

```bash
mkdir -p restaurant/kitchen/prep-area
```

Це створює три вкладені теки, навіть якщо `restaurant` та `kitchen` ще не існували.

---

## Команда 5: `touch` — «Створити порожній файл»

**touch** створює порожній файл. (Технічно вона оновлює мітку часу файлу, але якщо файл не існує — створює його.)

```bash
touch menu.txt
```

Перевірте:

```bash
ls
```

Ви побачите `menu.txt` у списку. Він порожній — просто чистий аркуш паперу, що чекає, поки на ньому напишуть.

---

## Команда 6: `cp` — «Скопіювати файл»

**cp** означає **C**o**p**y (копіювати). Вона створює дублікат файлу.

```bash
cp menu.txt menu-backup.txt
```

Тепер у вас два файли: оригінал і копія.

```bash
ls
```

Результат:
```
menu-backup.txt    menu.txt    my-first-folder    restaurant
```

Щоб скопіювати файл у теку:

```bash
cp menu.txt restaurant/
```

Щоб скопіювати цілу теку (з усім вмістом), використовуйте `-r` (від «recursive» — рекурсивно, тобто «цю теку й усе в ній»):

```bash
cp -r restaurant restaurant-copy
```

---

## Команда 7: `mv` — «Перемістити або перейменувати»

**mv** означає **M**o**v**e (перемістити). Вона робить дві речі:

**Переміщення файлу в іншу теку:**

```bash
mv menu-backup.txt restaurant/
```

Файлу тут більше немає — його переміщено в теку `restaurant`. На відміну від `cp`, оригінал не залишається.

**Перейменування файлу:**

```bash
mv menu.txt daily-specials.txt
```

Файл `menu.txt` зник. На його місці — `daily-specials.txt`. Той самий файл, нова назва. Переміщення й перейменування — це одна й та сама операція: ви просто змінюєте місцезнаходження (або назву) файлу.

---

## Команда 8: `rm` — «Викинути»

**rm** означає **R**e**m**ove (видалити). Вона видаляє файл.

```bash
rm daily-specials.txt
```

Файл зник.

### УВАГА: Кошика немає

Це найважливіше у всьому модулі:

> **`rm` не переміщує файли в кошик. Вона видаляє їх назавжди. Скасувати не можна. Немає запитання «Ви впевнені?». Файл зник.**

Уявіть це так: у графічному інтерфейсі видалення файлу переміщує його в Кошик. Ви можете відновити його. З `rm` файл знищується негайно.

**Щоб видалити теку з усім вмістом:**

```bash
rm -r restaurant-copy
```

Прапорець `-r` означає «recursive» (рекурсивно) — видалити цю теку й усе, що вона містить. Будьте дуже обережні з цим.

**Найнебезпечніша команда в комп'ютерній справі** (НЕ ВИКОНУЙТЕ це, просто знайте, що вона існує):

```
rm -rf /    ← НІКОЛИ ЦЕ НЕ РОБІТЬ. Видаляє все на комп'ютері. Абсолютно все.
```

Правило: завжди перевіряйте, що ви видаляєте, перш ніж натиснути Enter.

---

## Команда 9: `clear` — «Очистити екран»

Після виконання багатьох команд екран захаращується. `clear` очищує екран, щоб ви почали з чистого аркуша.

```bash
clear
```

Тепер ваш екран чистий. Нічого не видалено — старий вивід просто зсунувся вгору. Ви все ще можете прокрутити вгору, щоб його побачити.

**Клавіатурне скорочення**: На більшості терміналів `Ctrl + L` робить те саме.

## Коротка довідка

Тримайте під рукою, поки ці команди не стануть автоматичними:

| Команда | Що робить | Кухонна аналогія |
|---------|-----------|------------------|
| `pwd` | Показує, де ви | «В якій кімнаті я?» |
| `ls` | Показує, що тут є | «Що на цій полиці?» |
| `cd` | Переміщує в інше місце | «Перейти в іншу кімнату» |
| `mkdir` | Створює нову теку | «Побудувати нову кімнату» |
| `touch` | Створює порожній файл | «Покласти чистий аркуш на стіл» |
| `cp` | Копіює файл | «Зробити копію цього рецепту» |
| `mv` | Переміщує або перейменовує | «Перекласти на іншу полицю» або «змінити назву» |
| `rm` | Видаляє назавжди | «Знищити цей папір» (БЕЗ кошика!) |
| `clear` | Очищує екран | «Витерти дошку» |

## Чи знали ви?

- **Термінал старший за мишку на десятиліття.** Комп'ютери використовували лише текстові інтерфейси з 1960-х до середини 1980-х. Графічний інтерфейс із мишкою та вікнами, до якого ви звикли, був популяризований Apple Macintosh у 1984 році. Коли ви використовуєте термінал — ви використовуєте первісний спосіб спілкування людей із комп'ютерами.

- **`ls` — одна з найстаріших команд, що досі використовуються.** Вона з'явилась у 1961 році в Compatible Time-Sharing System (CTSS) MIT, де називалася `LISTF`. Сучасна `ls` з'явилася в першій версії Unix у 1971 році. Ви використовуєте команду, якій понад 50 років.

- **`~` (тильда) для домашньої директорії з'явилася через збіг на клавіатурі.** На ранніх терміналах клавіша Home і клавіша `~` були на одній фізичній кнопці. Умовність прижилася, і тепер кожен термінал у світі використовує `~` для позначення «домашня директорія».

## Бонус: З'єднання команд за допомогою каналів

Це **бонусний розділ** — можете переглянути його зараз і повернутися пізніше. Вам не обов'язково освоювати це сьогодні.

Іноді ви хочете взяти результат однієї команди й передати його іншій. Саме для цього є **канал** (pipe, символ `|`).

> Кухонна аналогія: Уявіть конвеєр. Одна станція нарізає овочі, потім передає їх наступній станції, яка їх готує. Кожна станція виконує одну роботу й передає результат далі.

Символ `|` (зазвичай знаходиться над клавішею Enter/Return, вводиться через Shift + зворотна коса) передає результат команди зліва до команди справа.

**Показати лише перші 5 файлів:**

```bash
ls | head -5
```

`ls` виводить усе, але `head -5` бере лише перші 5 рядків. Корисно, коли в теці сотні файлів.

**Шукати слово у файлі:**

```bash
cat menu.txt | grep "pasta"
```

`cat` відображає вміст файлу, а `grep "pasta"` фільтрує, показуючи лише рядки зі словом «pasta». (Ви будете використовувати `grep` ДУЖЕ часто у своїй кар'єрі.)

**Знайти команду, яку ви раніше вводили:**

```bash
history | grep "mkdir"
```

`history` показує кожну команду, яку ви вводили, а `grep "mkdir"` фільтрує лише ті, що містять «mkdir». Дуже зручно, коли ви не можете згадати точну команду.

Ви отримаєте більше практики з каналами в подальшому курсі. Наразі просто запам'ятайте: `|` з'єднує команди, як станції на конвеєрі.

## Типові помилки

| Помилка | Чому це проблема | Що робити натомість |
|---------|-----------------|---------------------|
| Використання `rm` без попередньої перевірки | Файли видаляються назавжди — без можливості скасування | Спочатку виконайте `ls`, щоб побачити, що ви збираєтесь видалити |
| Забули `-r` при копіюванні/видаленні тек | `cp folder newname` не спрацює для директорій | Використовуйте `cp -r folder newname` або `rm -r folder` |
| Пробіли в назвах файлів | `mkdir my folder` створює ДВІ теки: «my» та «folder» | Використовуйте лапки: `mkdir "my folder"` або дефіси: `mkdir my-folder` |
| Загубитися у файловій системі | Ви забуваєте, де знаходитесь, і створюєте файли не в тому місці | Часто вводьте `pwd`. Використовуйте `cd ~`, щоб повернутися додому, коли заблукали |
| Вводите команди неправильно й засмучуєтесь | Друкарські помилки трапляються у всіх, щодня | Натисніть стрілку вгору, щоб викликати останню команду й виправити її |

## Тест

1. **Що означає `pwd` і що ця команда робить?**
   <details>
   <summary>Відповідь</summary>
   pwd означає Print Working Directory (вивести робочу директорію). Вона показує повний шлях до директорії (теки), в якій ви зараз перебуваєте. У нашій кухонній аналогії це як запитати «в якій кімнаті я?»
   </details>

2. **Яка різниця між `cp` та `mv`?**
   <details>
   <summary>Відповідь</summary>
   `cp` (copy, копіювати) створює дублікат — оригінальний файл залишається на місці, а нова копія з'являється в місці призначення. `mv` (move, перемістити) переносить файл — він зникає з початкового місця й з'являється в новому. Уявіть це як ксерокопіювання проти фізичного переміщення аркуша паперу.
   </details>

3. **Чому `rm` небезпечніша за видалення файлів через графічний інтерфейс?**
   <details>
   <summary>Відповідь</summary>
   Коли ви видаляєте файл через графічний інтерфейс (Finder, File Explorer), він потрапляє в Кошик і може бути відновлений. Коли ви використовуєте `rm` у терміналі, файл видаляється назавжди й негайно. Немає кошика, немає можливості скасувати, немає підтвердження (якщо ви не додасте прапорець `-i`).
   </details>

4. **Як створити вкладену структуру тек на кшталт `project/src/components` однією командою?**
   <details>
   <summary>Відповідь</summary>
   Використовуйте `mkdir -p project/src/components`. Прапорець `-p` (від «parents» — батьківські) вказує mkdir створити всі батьківські директорії, яких ще немає. Без `-p` команда завершилася б помилкою, бо `project` та `project/src` ще не існують.
   </details>

5. **Ви загубилися у файловій системі. Які дві команди повернуть вас у безпечне місце?**
   <details>
   <summary>Відповідь</summary>
   `pwd` показує, де ви (щоб ви могли зорієнтуватися), а `cd ~` повертає вас у домашню директорію, де б ви не були. Символ `~` (тильда) — це скорочення, яке завжди означає «моя домашня директорія».
   </details>

## Практична вправа: Побудуйте структуру файлів ресторану

Попрактикуємо все вивчене, створивши файлову структуру для нашого уявного ресторану.

### Крок 1: Перейдіть у домашню директорію

```bash
cd ~
```

### Крок 2: Створіть структуру ресторану

```bash
mkdir -p restaurant/kitchen/prep-area
mkdir -p restaurant/kitchen/cooking-stations
mkdir -p restaurant/dining-room
mkdir -p restaurant/storage/pantry
mkdir -p restaurant/storage/freezer
```

### Крок 3: Створіть кілька файлів

```bash
touch restaurant/kitchen/prep-area/chopping-board.txt
touch restaurant/kitchen/cooking-stations/grill.txt
touch restaurant/kitchen/cooking-stations/oven.txt
touch restaurant/dining-room/table-1.txt
touch restaurant/dining-room/table-2.txt
touch restaurant/storage/pantry/flour.txt
touch restaurant/storage/pantry/sugar.txt
touch restaurant/storage/freezer/ice-cream.txt
```

### Крок 4: Подивіться, що ви побудували

```bash
ls restaurant/
ls restaurant/kitchen/
ls restaurant/kitchen/cooking-stations/
```

Очікуваний результат останньої команди:
```
grill.txt    oven.txt
```

### Крок 5: Перемістіть деякі речі

Морозиво тане! Перемістіть його з морозильника до зони підготовки:

```bash
mv restaurant/storage/freezer/ice-cream.txt restaurant/kitchen/prep-area/
```

Перевірте:

```bash
ls restaurant/kitchen/prep-area/
```

Очікуваний результат:
```
chopping-board.txt    ice-cream.txt
```

### Крок 6: Зробіть резервну копію меню

```bash
touch restaurant/menu.txt
cp restaurant/menu.txt restaurant/menu-backup.txt
ls restaurant/
```

### Крок 7: Прибирання

Коли закінчите експериментувати:

```bash
rm -r restaurant
```

Перевірте, що все видалено:

```bash
ls | grep restaurant
```

Відсутність результату означає, що теки більше немає.

**Критерії успіху**: Ви створили вкладену структуру директорій, створили файли всередині неї, перемістили файли між директоріями, скопіювали файл і прибрали за собою. Все це без жодного натискання кнопки миші.
