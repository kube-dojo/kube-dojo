## Чому цей модуль важливий

Все на вашому комп'ютері — кожна фотографія, кожна пісня, кожен додаток, кожне налаштування — зберігається як **файл**. І ці файли організовані в **директорії** (їх також називають теками).

Коли ви використовуєте GUI, ви бачите кольорові іконки тек і файлів. Ви двічі клацаєте, щоб відкрити їх. Але за лаштунками ваш комп'ютер уявляє файли й директорії як дерево текстових шляхів, на кшталт `/home/yourname/Documents/report.txt`.

У Kubernetes та хмарній інженерії ви постійно працюватимете з файлами: файлами конфігурації, скриптами, логами, маніфестами. Якщо ви не вмієте орієнтуватися у файлах і директоріях із терміналу — ви загубитесь. Цей модуль це виправить.

## Що таке файл?

**Файл** — це контейнер для інформації, що зберігається на вашому комп'ютері.

Уявіть його як аркуш паперу з написаним текстом. Аркуш має:

- **Назву** (щоб його знайти): `grocery-list.txt`
- **Вміст** (власне інформація): «яйця, молоко, хліб»
- **Розташування** (де він зберігається): на столі, у шухляді, у шафі

Файли можуть містити що завгодно:
- Текст (як нотатку або конфігурацію)
- Код (інструкції для комп'ютера)
- Зображення (ваші фотографії)
- Музику, відео, бази даних і багато іншого

> У нашій аналогії з кухнею ресторану: файл — це **картка рецепту**. Вона має назву («Томатний суп»), вміст (власне рецепт) і зберігається десь (у коробці з рецептами, на полиці, у шухляді).

### Все є файлом (справді!)

Ось дещо дивовижне про Linux (операційну систему, на якій працює Kubernetes):

**Майже все представлено як файл.**

- Ваші фактичні файли? Файли.
- Ваша клавіатура? Система бачить її як файл (`/dev/input/event0`).
- Ваш жорсткий диск? Файл (`/dev/sda`).
- Запущені програми? Вони мають записи в `/proc/`, що виглядають як файли.

Вам не потрібно зараз турбуватися про файли пристроїв. Просто знайте, що концепція «файлів» у Linux значно глибша, ніж документи й фотографії. Ця філософія проєктування — одна з причин, чому Linux настільки потужний: все можна читати, записувати та маніпулювати за допомогою одного й того самого набору інструментів.

## Що таке директорія?

**Директорія** — це контейнер для файлів (та інших директорій). Це те саме, що **тека** — буквально та сама концепція, просто інша назва.

- Користувачі GUI кажуть «тека» (бо іконка схожа на паперову теку).
- Користувачі терміналу кажуть «директорія» (технічний термін).

> На кухні: директорія — це **шухляда** або **полиця**. Ваші картки рецептів (файли) зберігаються в шухлядах (директоріях). Шухляди можуть містити інші шухляди (піддиректорії).

---

## Дерево директорій

Ваш комп'ютер організовує всі файли та директорії в **деревоподібну** структуру. Вона починається з однієї точки вгорі й розгалужується вниз.

```
/                          <-- "Корінь" — найвища точка, перший поверх
├── home/                  <-- Де живуть облікові записи користувачів
│   └── yourname/          <-- ВАША домашня директорія
│       ├── Documents/
│       ├── Downloads/
│       ├── Desktop/
│       └── projects/
├── etc/                   <-- Файли конфігурації системи
├── var/                   <-- Змінні дані (логи, бази даних)
├── tmp/                   <-- Тимчасові файли
└── usr/                   <-- Програми та утиліти користувача
```

### Ключові розташування

| Шлях | Що це | Кухонна аналогія |
|------|-------|------------------|
| `/` | **Коренева директорія** — сама верхівка дерева | Сама будівля — все знаходиться всередині |
| `/home/yourname/` | **Ваша домашня директорія** — ваш особистий простір | Ваше особисте робоче місце на кухні |
| `~` | **Скорочення для домашньої директорії** | Прізвисько вашого робочого місця |
| `/etc/` | Файли конфігурації системи | Книга правил ресторану та стандарти рецептів |
| `/tmp/` | Тимчасові файли (видаляються при перезавантаженні) | Стіл для підготовки — використовується під час готування, прибирається після |
| `/var/log/` | Файли логів (записи подій) | Книга історії замовлень |

> Символ `~` (тильда) — це скорочення. Замість введення `/home/yourname/` можна просто ввести `~`. Ваш термінал зрозуміє. Це як мати прізвисько — простіше використовувати, ніж повне ім'я.

### На macOS

macOS трохи відрізняється. Ваша домашня директорія знаходиться за шляхом `/Users/yourname/` замість `/home/yourname/`. Але `~` все одно працює як скорочення, тому вам рідко потрібно про це думати.

## Навігація: Де я?

Коли ви відкриваєте термінал, ви «стоїте» в директорії — зазвичай у домашній. Термінал не показує візуальну карту; потрібно запитати.

### `pwd` — Вивести робочу директорію

«Де я зараз?»

```bash
$ pwd
/home/yourname
```

`pwd` означає **Print Working Directory**. Показує повний шлях до місця, де ви зараз.

> Кухонна аналогія: «В якій кімнаті я стою?» — pwd відповідає на це.

### `ls` — Показати вміст

«Що є в цій кімнаті?»

```bash
$ ls
Desktop    Documents    Downloads    Music    Pictures
```

`ls` означає **list** (список). Показує файли та директорії у вашому поточному розташуванні.

Можна додати параметри для більш детального виводу:

```bash
$ ls -l
```

Прапорець `-l` означає «long format» (довгий формат). Тепер ви побачите щось на кшталт:

```
drwxr-xr-x  2 yourname yourname 4096 Mar 23 10:00 Desktop
drwxr-xr-x  3 yourname yourname 4096 Mar 23 09:45 Documents
-rw-r--r--  1 yourname yourname  220 Mar 23 08:30 notes.txt
```

Не панікуйте — скоро ми це розшифруємо.

### `cd` — Змінити директорію

«Перейти в іншу кімнату.»

```bash
$ cd Documents
$ pwd
/home/yourname/Documents
```

`cd` означає **change directory** (змінити директорію). Ви вказуєте, куди йти, і команда переносить вас туди.

Деякі важливі скорочення `cd`:

| Команда | Куди переносить | Аналогія |
|---------|----------------|----------|
| `cd ~` або просто `cd` | Ваша домашня директорія | «Повернутися на моє робоче місце» |
| `cd ..` | На один рівень вгору (батьківська директорія) | «Перейти в кімнату, що містить цю» |
| `cd -` | Остання директорія, де ви були | «Повернутися туди, де я щойно був» |
| `cd /` | Коренева директорія | «Перейти на перший поверх» |

## Абсолютні та відносні шляхи

Це важлива концепція. Є два способи описати, де знаходиться файл або директорія:

### Абсолютний шлях (повна адреса)

Абсолютний шлях починається від кореня (`/`) і дає повне розташування:

```
/home/yourname/Documents/report.txt
```

Це як давати повну поштову адресу: **вул. Хрещатик, 123, Київ, Україна**. Де б ви не були, ця адреса вказує на одне конкретне місце.

### Відносний шлях (напрямок звідси)

Відносний шлях описує розташування відносно того, де ви зараз:

```
Documents/report.txt
```

Це як сказати: «Два квартали ліворуч, потім один вгору.» Ці вказівки працюють лише якщо ви знаєте початкову точку.

### Приклад

```bash
$ pwd
/home/yourname

# Ці дві команди роблять одне й те саме:
$ cd /home/yourname/Documents       # Абсолютний шлях
$ cd Documents                       # Відносний шлях (працює, бо ми в /home/yourname)
```

### Спеціальні символи шляхів

| Символ | Значення | Приклад |
|--------|----------|---------|
| `/` | Коренева директорія (на початку) або розділювач шляху | `/home/yourname` |
| `~` | Ваша домашня директорія | `~/Documents` = `/home/yourname/Documents` |
| `.` | Поточна директорія («тут») | `./script.sh` = «script.sh у цій директорії» |
| `..` | Батьківська директорія («на рівень вище») | `../Downloads` = «піднятися вгору, потім у Downloads» |

## Читання файлів

Тепер заглянемо всередину файлів. У вас є кілька інструментів, кожен корисний у різних ситуаціях:

### `cat` — Показати весь файл

```bash
$ cat notes.txt
This is my first note.
I wrote it in the terminal!
```

`cat` означає «concatenate» (об'єднання), але більшість людей використовують цю команду для відображення вмісту файлу. Вона виводить весь файл на екран.

> Кухонна аналогія: `cat` — це як прочитати всю картку рецепту вголос від початку до кінця.

**Коли використовувати**: Для невеликих файлів (менше 50 рядків).

### `head` — Показати перші 10 рядків

```bash
$ head long-file.txt
```

Показує лише перші 10 рядків. Можна змінити кількість:

```bash
$ head -n 5 long-file.txt    # Показати перші 5 рядків
```

> Кухонна аналогія: «Прочитай мені лише назву та інгредієнти — повний рецепт не потрібен.»

### `tail` — Показати останні 10 рядків

```bash
$ tail log-file.txt
```

Показує останні 10 рядків. Надзвичайно корисно для читання файлів логів, де найновіші записи внизу.

```bash
$ tail -n 20 log-file.txt    # Показати останні 20 рядків
```

> Кухонна аналогія: «Які останні кілька замовлень надійшли?»

## Створення файлів і директорій

### `mkdir` — Створити директорію

```bash
$ mkdir recipes
```

Створює нову директорію з назвою `recipes` у вашому поточному розташуванні.

Щоб створити вкладені директорії (директорію всередині директорії всередині директорії):

```bash
$ mkdir -p recipes/italian/pasta
```

Прапорець `-p` означає «створити батьківські директорії за потреби». Без нього ви отримали б помилку, якби `recipes/` або `recipes/italian/` ще не існували.

### `touch` — Створити порожній файл

```bash
$ touch shopping-list.txt
```

Створює новий порожній файл. (Якщо файл вже існує, оновлює його мітку часу без зміни вмісту.)

> Кухонна аналогія: `mkdir` — побудувати нову шухляду. `touch` — покласти в неї порожню картку рецепту.

## Приховані файли (дотфайли)

Деякі файли та директорії починаються з крапки (`.`). Їх називають **прихованими файлами** або **дотфайлами**.

```bash
$ ls
Documents    Downloads    Music

$ ls -a
.   ..   .bashrc   .config   Documents   Downloads   Music
```

Прапорець `-a` означає «all» (усе) — включно з прихованими файлами.

Приховані файли зазвичай містять конфігурації та налаштування. Деякі поширені:

| Файл | Що він робить |
|------|---------------|
| `.bashrc` | Налаштування вашого терміналу bash |
| `.zshrc` | Налаштування вашого терміналу zsh (стандарт macOS) |
| `.config/` | Директорія з конфігураціями додатків |
| `.ssh/` | Ключі SSH (для безпечних з'єднань) |
| `.gitconfig` | Налаштування Git |

> Ці файли приховані, бо вам не потрібно бачити їх щодня, а випадкове їх видалення може зіпсувати налаштування. Вони як електропроводка за стінами кухні — важлива, але зазвичай прихована від очей.

## Основи дозволів на файли

Пам'ятаєте, коли ми виконали `ls -l` і побачили таке?

```
drwxr-xr-x  2 yourname yourname 4096 Mar 23 10:00 Desktop
-rw-r--r--  1 yourname yourname  220 Mar 23 08:30 notes.txt
```

Розшифруймо перший стовпець: `-rw-r--r--`

### Рядок дозволів

```
-  rw-  r--  r--
|  |    |    |
|  |    |    └── Дозволи для інших (усіх решта)
|  |    └── Дозволи групи
|  └── Дозволи власника (вас)
└── Тип файлу (- = файл, d = директорія)
```

### Три типи дозволів

| Літера | Дозвіл | Для файлів | Для директорій |
|--------|--------|------------|----------------|
| `r` | Читання (read) | Можна бачити вміст | Можна переглянути, що всередині |
| `w` | Запис (write) | Можна змінювати вміст | Можна додавати або видаляти файли |
| `x` | Виконання (execute) | Можна запустити як програму | Можна увійти в директорію (`cd` в неї) |
| `-` | Немає дозволу | Не можна виконати цю дію | Не можна виконати цю дію |

### Читаємо приклад

```
-rw-r--r--  notes.txt
```

- `-` : Це звичайний файл (не директорія)
- `rw-` : Власник може **читати** та **записувати** (але не виконувати)
- `r--` : Група може лише **читати**
- `r--` : Інші можуть лише **читати**

```
drwxr-xr-x  Desktop
```

- `d` : Це **директорія**
- `rwx` : Власник може читати, записувати та входити
- `r-x` : Група може читати та входити (але не додавати/видаляти файли)
- `r-x` : Інші можуть читати та входити (але не додавати/видаляти файли)

> Кухонна аналогія: Дозволи — це як **хто має який ключ**. Шеф-кухар (власник) має ключ до всього. Су-шефи (група) можуть відкрити більшість шухляд. Офіціанти (інші) можуть лише зазирнути через вікно.

Не турбуйтесь поки що про зміну дозволів — просто знайте, як їх читати. Ми розглянемо `chmod`, коли він вам знадобиться.

## Чи знали ви?

> 1. **Коренева директорія `/` називається «root» (корінь), бо дерево директорій росте вниз.** Як справжнє дерево, корінь знаходиться вгорі, а гілки простягаються вниз. Кожен файл на вашому комп'ютері знаходиться десь на гілці, що з'єднується назад до `/`.
>
> 2. **Скорочення `~` було обрано через розкладку клавіатури.** На терміналі ADM-3A (виготовленому у 1970-х, одному з перших відеотерміналів) клавіша Home і символ `~` були на одній фізичній клавіші. Тому `~` стала скороченням для домашньої директорії, і ця умовність тримається вже понад 50 років.
>
> 3. **Linux сприймає все як файл — включно з вашою клавіатурою.** Коли ви натискаєте клавішу, ядро записує це натискання у файлоподібний інтерфейс. Програми зчитують з цього інтерфейсу, щоб дізнатися, що ви ввели. Ця філософія «все є файлом» — причина, чому Linux можна повністю контролювати з терміналу: завжди є файл, який можна прочитати або записати.

## Типові помилки

| Помилка | Що трапляється | Виправлення |
|---------|---------------|-------------|
| `cd` у файл замість директорії | Помилка `Not a directory` | Використовуйте `cat` або `head` для читання файлів; `cd` — для директорій |
| Забули пробіл між командою та шляхом | `cdDocuments: command not found` | Завжди ставте пробіл: `cd Documents` |
| Зворотні скісні `\` замість прямих `/` | Шлях не знайдено | Linux/macOS використовує прямі скісні: `/home/you/Documents` |
| Створили файл, коли хотіли директорію | Ви отримуєте файл з назвою «recipes» замість теки | Використовуйте `mkdir` для директорій, `touch` для файлів |
| Загубились у вкладених директоріях | Не уявляєте, де ви | Виконайте `pwd`, щоб побачити своє розташування, або `cd ~`, щоб повернутися додому |
| Друкарська помилка в назві шляху | `No such file or directory` | Спочатку використовуйте `ls`, щоб побачити, що є, потім вводьте обережно (або використовуйте автодоповнення Tab — натисніть **Tab** для автоматичного доповнення назв!) |

> **Порада: Автодоповнення Tab.** Почніть вводити назву файлу або директорії й натисніть **Tab**. Термінал автоматично доповнить її. Якщо є кілька варіантів, натисніть **Tab** двічі, щоб побачити всі можливості. Це економить введення та запобігає помилкам.

## Тест

**Запитання 1**: Яка команда показує ваше поточне розташування в дереві директорій?

<details>
<summary>Показати відповідь</summary>

```bash
$ pwd
```

Означає **Print Working Directory** і показує повний шлях до місця, де ви зараз.

</details>

**Запитання 2**: Що означає `~`?

<details>
<summary>Показати відповідь</summary>

Вашу **домашню директорію**. Це скорочення, щоб не вводити повний шлях (на кшталт `/home/yourname/`).

</details>

**Запитання 3**: Яка різниця між `ls` та `ls -a`?

<details>
<summary>Показати відповідь</summary>

`ls` показує лише видимі файли та директорії. `ls -a` показує **всі** файли, включно з прихованими (дотфайлами), що починаються з символу `.`.

</details>

**Запитання 4**: Якщо ви бачите `-rw-r-----` на файлі, хто може його читати?

<details>
<summary>Показати відповідь</summary>

- **Власник** може читати й записувати (`rw-`).
- **Група** може лише читати (`r--`).
- **Інші** не мають дозволів (`---`).

Отже, власник та члени групи файлу можуть його читати. Усім решта доступ заблоковано.

</details>

**Запитання 5**: Яка різниця між `/home/yourname/Documents` та `Documents`?

<details>
<summary>Показати відповідь</summary>

`/home/yourname/Documents` — це **абсолютний шлях**: він працює звідусіль, бо починається від кореня (`/`).

`Documents` — це **відносний шлях**: він працює лише якщо ви зараз у `/home/yourname/`.

</details>

## Практична вправа: Побудуйте свою першу структуру директорій

### Мета

Створити структуру директорій, додати файли, прочитати їх і перевірити дозволи — все з терміналу.

### Кроки

1. **Перейдіть у домашню директорію:**

```bash
$ cd ~
$ pwd
```

Переконайтесь, що бачите шлях до своєї домашньої директорії.

2. **Створіть структуру проєкту:**

```bash
$ mkdir -p kubedojo-practice/recipes/appetizers
$ mkdir -p kubedojo-practice/recipes/main-courses
$ mkdir -p kubedojo-practice/recipes/desserts
```

3. **Перейдіть усередину:**

```bash
$ cd kubedojo-practice
$ ls
```

Ви маєте побачити: `recipes`

4. **Створіть кілька файлів:**

```bash
$ touch recipes/appetizers/bruschetta.txt
$ touch recipes/main-courses/pasta-carbonara.txt
$ touch recipes/desserts/tiramisu.txt
```

5. **Додайте вміст до файлу** (ми використаємо `echo` з `>` для запису у файл):

```bash
$ echo "Ingredients: bread, tomatoes, basil, olive oil" > recipes/appetizers/bruschetta.txt
$ echo "Ingredients: pasta, eggs, pancetta, parmesan" > recipes/main-courses/pasta-carbonara.txt
$ echo "Ingredients: coffee, mascarpone, ladyfingers, cocoa" > recipes/desserts/tiramisu.txt
```

> Символ `>` означає «надіслати вивід у цей файл» замість відображення на екрані. Уявіть це як перенаправлення відповіді шеф-кухаря: замість того щоб сказати вголос — записати на картку рецепту.

6. **Прочитайте файли:**

```bash
$ cat recipes/appetizers/bruschetta.txt
$ head recipes/main-courses/pasta-carbonara.txt
$ tail recipes/desserts/tiramisu.txt
```

7. **Перевірте дозволи:**

```bash
$ ls -l recipes/appetizers/
```

Ви маєте побачити рядок дозволів для вашого файлу bruschetta.

8. **Перевірте приховані файли у домашній директорії:**

```bash
$ ls -a ~
```

Шукайте файли, що починаються з `.` — це ваші дотфайли!

9. **Поміщуйтесь навколо:**

```bash
$ cd recipes/desserts
$ pwd                          # Де ви?
$ cd ..                        # Піднятися на рівень
$ pwd                          # А тепер де?
$ cd ~                         # Повернутися додому
$ pwd                          # Знову вдома
```

### Критерії успіху

Ви виконали цю вправу, коли можете:

- [ ] Перейти до домашньої директорії через `cd ~`
- [ ] Створити вкладені директорії через `mkdir -p`
- [ ] Створити файли через `touch`
- [ ] Записати вміст у файли через `echo "text" > file`
- [ ] Прочитати файли через `cat`, `head` і `tail`
- [ ] Перевірити дозволи файлів через `ls -l`
- [ ] Побачити приховані файли через `ls -a`
- [ ] Переміщуватись через `cd`, `cd ..` та `cd ~`
