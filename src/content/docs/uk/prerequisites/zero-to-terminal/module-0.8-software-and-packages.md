## Чому цей модуль важливий

Щоб працювати з Kubernetes, Docker та хмарними інструментами, вам потрібно буде **встановлювати програми** на комп'ютер — не завантаженням інсталяторів із вебсайтів і натисканням «Далі, Далі, Готово», а введенням однієї команди в терміналі.

Цей модуль навчить вас, як працює програмне забезпечення, що таке менеджери пакетів і як встановити перші інструменти з командного рядка. Це навички, які ви використовуватимете буквально щодня в інженерії.

## Що таке програмне забезпечення?

Почнімо з самого початку.

**Програмне забезпечення** — це набір інструкцій, що вказує комп'ютеру, що робити. Коли ви відкриваєте веббраузер, відтворюєте відео або виконуєте команду в терміналі — програмне забезпечення робить це можливим.

Програми пишуть люди **мовами програмування** — такими як Python, Go, Java або JavaScript, які розроблені так, щоб їх могли читати і люди, і комп'ютери (більш-менш).

Ось крихітний приклад на Python:

```python
print("Hello, World!")
```

Це програмне забезпечення. Один рядок, що каже комп'ютеру: «Виведи текст Hello, World! на екран.»

> Кухонна аналогія: Програмне забезпечення — це **рецепт**. Це інструкції з приготування страви. Комп'ютер — це шеф-кухар, який слідує рецепту саме так, як написано. Мови програмування — це мова, якою написаний рецепт (українська, англійська тощо).

## Від вихідного коду до працюючої програми

Кожна програма проходить шлях від «слів, набраних людиною» до «чогось, що комп'ютер може запустити»:

### Крок 1: Вихідний код

Це те, що пишуть програмісти. Виглядає як текст:

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello from Go!")
}
```

Ви можете його прочитати (більш-менш). Ваш комп'ютер не може запустити його безпосередньо.

### Крок 2: Компіляція (для деяких мов)

Деякі мови потрібно **скомпілювати** — перекласти з людиночитаного коду в **машинний код** (двійковий — одиниці й нулі, які розуміє процесор комп'ютера).

```
Вихідний код  →  Компілятор  →  Двійковий файл (виконуваний)
(рецепт)         (перекладач)   (готова страва, подана на тарілці)
```

Результат називається **двійковий файл** (binary) або **виконуваний файл** (executable) — файл, який комп'ютер може запустити.

> Кухонна аналогія: Вихідний код — це рецепт на папері. Компіляція — це процес приготування. Двійковий файл — готова страва, сервірована й готова до подачі.

### Крок 3: Виконання

Ви **запускаєте** (виконуєте) двійковий файл, і комп'ютер слідує інструкціям.

```bash
$ ./my-program
Hello from Go!
```

> Не всі мови потребують компіляції. Python, наприклад, є **інтерпретованою** мовою — вона читає й виконує код рядок за рядком, як шеф-кухар, що читає рецепт крок за кроком під час готування. Мови на кшталт Go, C та Rust спочатку компілюються, потім запускаються — як шеф-кухар, що заздалегідь готує всі інгредієнти.

## Що таке пакет?

Встановлювати програми з вихідного коду складно. Вам потрібно було б:

1. Завантажити вихідний код
2. Встановити потрібний компілятор
3. Скомпілювати
4. Перемістити двійковий файл у правильне місце
5. Сподіватися, що нічого не пішло не так

**Пакет** загортає все це в акуратний набір. Це вихідний код, вже скомпільований (зазвичай), упакований з інструкціями, куди встановити й що ще потрібно.

> Кухонна аналогія: Пакет — це **набір для приготування страви** (як Blue Apron чи HelloFresh). Замість того щоб іти в магазин, шукати кожен інгредієнт і вираховувати пропорції, хтось зібрав усе разом для вас. Просто відкрийте коробку й слідуйте простим інструкціям.

## Що таке менеджер пакетів?

**Менеджер пакетів** — це інструмент, що завантажує, встановлює, оновлює та видаляє пакети за вас. Це як **магазин додатків** для вашого терміналу.

Замість відвідування вебсайту, завантаження файлу та клацання через інсталятор ви вводите одну команду:

```bash
$ sudo apt install htop       # На Ubuntu/Debian Linux
$ brew install htop            # На macOS
```

І менеджер пакетів:
1. Знаходить пакет у своєму каталозі
2. Завантажує його
3. Встановлює
4. Налаштовує, щоб ви могли ним користуватися

### Поширені менеджери пакетів

| Менеджер пакетів | Операційна система | Команда встановлення |
|-----------------|-------------------|---------------------|
| **apt** | Ubuntu, Debian (Linux) | `sudo apt install package-name` |
| **dnf** / **yum** | Fedora, RHEL, CentOS (Linux) | `sudo dnf install package-name` |
| **brew** (Homebrew) | macOS (та Linux) | `brew install package-name` |
| **pacman** | Arch Linux | `sudo pacman -S package-name` |
| **choco** | Windows | `choco install package-name` |

> У цьому курсі ми здебільшого використовуватимемо **apt** (для Linux) та **brew** (для macOS), бо вони найпоширеніші у світі Kubernetes.

## Що таке `sudo`?

Ви помітите, що деякі команди починаються з `sudo`. Це важливо.

**`sudo`** означає **«superuser do»** (виконати від суперкористувача) — запускає команду з **привілеями адміністратора**.

Ваш комп'ютер має систему безпеки: звичайні користувачі не можуть встановлювати програми системно, змінювати системні файли чи робити щось, що може зламати комп'ютер. Це зроблено навмисне. Це запобігає випадковостям і забезпечує безпеку системи.

Але встановлення програм вимагає запису файлів у системні директорії, до яких звичайні користувачі не мають доступу. Тому ви використовуєте `sudo`, щоб тимчасово стати **суперкористувачем** (також називають **root** — всемогутній обліковий запис адміністратора).

```bash
$ apt install htop              # Доступ заборонено
$ sudo apt install htop         # Працює! (запитає пароль)
```

Коли ви вводите `sudo`, у вас запитають пароль. Це ваш пароль користувача — той самий, що ви використовуєте для входу. Коли ви його вводите, на екрані не з'являються жодні символи (ні крапки, ні зірочки, нічого). Це нормально й зроблено навмисне — це заважає тому, хто дивиться через ваше плече, підрахувати кількість символів. Просто введіть пароль і натисніть Enter.

> Кухонна аналогія: `sudo` — це як **ключ менеджера**. Більшість персоналу може працювати на кухні, але щоб увійти до кладової або змінити налаштування термостата — потрібен ключ менеджера. `sudo` дає вам цей ключ тимчасово.

### На macOS з Homebrew

Homebrew (`brew`) розроблений так, що зазвичай **не потрібен `sudo`**. Він встановлює пакети у ваш користувацький простір, а не в системні директорії. Це одна з причин популярності Homebrew — менше метушні з дозволами.

```bash
$ brew install htop             # Працює без sudo на macOS
```

## Залежності: Програми, яким потрібні інші програми

Програми рідко працюють самостійно. Більшість потребують інших програм або бібліотек для роботи. Їх називають **залежностями**.

Наприклад:
- Вебзастосунок може залежати від бази даних
- Інструмент командного рядка може залежати від конкретної бібліотеки
- Програма на Python залежить від встановленого Python

> Кухонна аналогія: Залежності — це як **інгредієнти для інгредієнтів**. Щоб зробити особливий соус, потрібен майонез. Але щоб зробити майонез, потрібні яйця та олія. Яйця й олія — це залежності майонезу, який сам є залежністю особливого соусу.

### Чому залежності важливі

**Добра новина**: Менеджери пакетів обробляють залежності автоматично. Коли ви встановлюєте пакет, менеджер також встановлює все, що цьому пакету потрібно.

```bash
$ sudo apt install some-program
Reading package lists... Done
The following additional packages will be installed:
  dependency-1 dependency-2 dependency-3
```

Менеджер пакетів розбирається з усім ланцюжком залежностей і встановлює їх усіх. Вам не потрібно шукати їх самостійно.

**Менш приємна новина**: Іноді залежності конфліктують між собою. Програма A потребує версію 1.0 бібліотеки, а Програма B — версію 2.0. Це називається **пекло залежностей** <!-- VERIFY: пекло залежностей -->, і це одна з проблем, для розв'язання якої були винайдені контейнери (про які ви скоро дізнаєтесь).

## Встановлення перших пакетів

Встановімо кілька корисних інструментів. Слідкуйте за інструкціями для вашої операційної системи.

### Оновлення списку пакетів

Перед встановленням чого-небудь оновіть каталог менеджера пакетів. Уявіть це як оновлення списку доступних товарів:

**Ubuntu/Debian Linux:**

```bash
$ sudo apt update
```

Це нічого не встановлює й не змінює — лише завантажує найновіший список доступних пакетів та їхніх версій.

**macOS:**

Спочатку, якщо у вас ще не встановлений Homebrew, встановіть його зараз:

```bash
# First, install Homebrew (macOS only — skip if you already have it)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

> Це може зайняти кілька хвилин. Попросять ввести пароль (той, що ви використовуєте для входу на Mac).

Коли Homebrew встановлено, оновіть його:

```bash
$ brew update
```

### Встановлення `htop` — системного монітора

`htop` — це візуальний інструмент, що показує, які програми запущені на вашому комп'ютері, скільки CPU та пам'яті вони використовують, і багато іншого.

**Ubuntu/Debian Linux:**

```bash
$ sudo apt install htop
```

**macOS:**

```bash
$ brew install htop
```

Тепер запустіть:

```bash
$ htop
```

Ви побачите кольоровий дисплей, що показує використання CPU, пам'яті та запущені процеси (програми). Це як дивитися на дошку замовлень кухні — ви бачите все, що відбувається одразу.

**Натисніть `q`, щоб вийти з htop.**

### Встановлення `tree` — візуалізатора директорій

Пам'ятаєте, як ми створювали директорії в Модулі 0.4? `tree` показує структуру директорій у красивому візуальному форматі.

**Ubuntu/Debian Linux:**

```bash
$ sudo apt install tree
```

**macOS:**

```bash
$ brew install tree
```

Тепер спробуйте:

```bash
$ tree ~/kubedojo-practice
```

Ви маєте побачити щось на кшталт:

```
/home/yourname/kubedojo-practice
└── recipes
    ├── appetizers
    │   └── bruschetta.txt
    ├── desserts
    │   └── tiramisu.txt
    └── main-courses
        └── pasta-carbonara.txt
```

(Якщо ви виконали вправу з Модуля 0.4. Якщо ні, `tree` все одно працює — просто спробуйте на будь-якій директорії.)

## Оновлення та видалення програм

### Оновлення всіх встановлених пакетів

З часом програми на вашому комп'ютері отримують оновлення — виправлення помилок, патчі безпеки, нові функції. Оновлюватися варто регулярно.

**Ubuntu/Debian Linux:**

```bash
$ sudo apt update              # Оновити список пакетів
$ sudo apt upgrade             # Встановити доступні оновлення
```

Можна об'єднати:

```bash
$ sudo apt update && sudo apt upgrade
```

`&&` означає «виконати другу команду лише якщо перша успішна». Уявіть це як: «Оновити список І ПОТІМ встановити оновлення.»

**macOS:**

```bash
$ brew update && brew upgrade
```

### Видалення програм

**Ubuntu/Debian Linux:**

```bash
$ sudo apt remove package-name
```

**macOS:**

```bash
$ brew uninstall package-name
```

### Пошук пакетів

Не впевнені, як називається пакет?

**Ubuntu/Debian Linux:**

```bash
$ apt search keyword
```

**macOS:**

```bash
$ brew search keyword
```

## Чи знали ви?

> 1. **Homebrew (менеджер пакетів для macOS) був створений у 2009 році розробником, якого дратувало, що macOS не має нормального менеджера пакетів.** Макс Хоуелл створив його як проєкт з відкритим кодом. Сьогодні він має понад 6 000 пакетів і використовується мільйонами розробників. Назва — метафора пивоваріння: пакети називаються «formulae» (рецептури), місце встановлення — «Cellar» (погріб), а вся система «варить» (brews) ваше програмне забезпечення.
>
> 2. **Менеджер пакетів `apt` на Ubuntu має доступ до понад 60 000 пакетів.** Це 60 000 програм, які можна встановити однією командою. Від текстових редакторів до баз даних, від ігор до інструментів наукових обчислень — це один із найбільших каталогів програм у світі, і все безплатно.
>
> 3. **Концепція `sudo` виникла з реальної потреби безпеки.** У 1980 році програмісти з SUNY Buffalo потребували спосіб дозволити довіреним користувачам виконувати конкретні команди від імені root без розголошення пароля root. Вони створили `sudo` — спочатку як скорочення від «superuser do». Система логує кожну команду `sudo`, тому адміністратори можуть перевірити, хто що робив. Сьогодні `sudo` використовується практично на кожній системі Linux та macOS.

## Типові помилки

| Помилка | Що трапляється | Виправлення |
|---------|---------------|-------------|
| Забули `sudo` на Linux | `Permission denied` або `Operation not permitted` | Додайте `sudo` перед командою: `sudo apt install ...` |
| Використовуєте `sudo` з `brew` на macOS | Homebrew попереджає або встановлює неправильно | Не використовуйте `sudo` з `brew` — він його не потребує |
| Не виконали `apt update` спочатку | Можете встановити стару версію або не знайти пакет | Завжди виконуйте `sudo apt update` перед встановленням на Linux |
| Друкарська помилка в назві пакета | `Unable to locate package htoop` | Перевірте написання або використайте `apt search` / `brew search`, щоб знайти правильну назву |
| Не читаєте вивід | Пропускаєте важливі попередження чи помилки | Читайте, що вам каже термінал! Він часто пояснює, що саме пішло не так |
| Натискаєте Enter під час запиту пароля, не ввівши нічого | Помилка автентифікації | Введіть пароль (символи не відображатимуться) і натисніть Enter |

## Тест

**Запитання 1**: Що таке менеджер пакетів?

<details>
<summary>Показати відповідь</summary>

Інструмент, що автоматично завантажує, встановлює, оновлює та видаляє програмні пакети. Це як магазин додатків для терміналу. Приклади: `apt` (Ubuntu/Debian), `brew` (macOS), `dnf` (Fedora).

</details>

**Запитання 2**: Що означає `sudo` і навіщо він потрібен?

<details>
<summary>Показати відповідь</summary>

`sudo` означає **«superuser do»** (виконати від суперкористувача). Він потрібен, бо встановлення програм системно вимагає привілеїв адміністратора (root). Звичайні користувачі не можуть записувати в системні директорії з міркувань безпеки. `sudo` тимчасово підвищує ваші дозволи.

</details>

**Запитання 3**: Що таке залежність?

<details>
<summary>Показати відповідь</summary>

Залежність — це програма, яка потрібна іншій програмі для роботи. Наприклад, якщо Програма A потребує Бібліотеку B, то Бібліотека B — це залежність Програми A. Менеджери пакетів обробляють залежності автоматично.

</details>

**Запитання 4**: Яка різниця між `apt update` та `apt upgrade`?

<details>
<summary>Показати відповідь</summary>

- `apt update` оновлює список доступних пакетів та їхніх версій (завантажує найновіший каталог). Нічого не встановлює й не змінює.
- `apt upgrade` фактично встановлює доступні оновлення для ваших встановлених пакетів.

Потрібно виконати `update` перед `upgrade`, щоб система знала, які оновлення доступні.

</details>

**Запитання 5**: Чому ви не бачите символів при введенні пароля після `sudo`?

<details>
<summary>Показати відповідь</summary>

Це функція безпеки. Не показуючи жодних символів (навіть крапок чи зірочок), система запобігає тому, щоб хтось, хто дивиться на ваш екран, побачив довжину вашого пароля. Просто введіть пароль і натисніть Enter — він приймається, навіть якщо нічого не з'являється.

</details>

## Практична вправа: Ваші перші встановлення програм

### Мета

Використати менеджер пакетів для встановлення, запуску та дослідження нових програм із терміналу.

### Кроки

1. **Оновіть менеджер пакетів:**

На Ubuntu/Debian Linux:
```bash
$ sudo apt update
```

На macOS:
```bash
$ brew update
```

2. **Встановіть htop:**

На Ubuntu/Debian Linux:
```bash
$ sudo apt install htop -y
```

На macOS:
```bash
$ brew install htop
```

Прапорець `-y` (для apt) означає «так на всі запитання» — автоматично підтверджує встановлення без запитання «Ви впевнені? [Y/n]».

3. **Запустіть htop і досліджуйте:**

```bash
$ htop
```

Зверніть увагу на:
- Індикатори використання CPU вгорі
- Індикатор використання пам'яті
- Список запущених процесів
- Кожен процес має PID (Process ID — унікальний номер)

Натисніть `q` для виходу.

4. **Встановіть tree:**

На Ubuntu/Debian Linux:
```bash
$ sudo apt install tree -y
```

На macOS:
```bash
$ brew install tree
```

5. **Використайте tree для візуалізації директорії:**

```bash
$ tree ~/kubedojo-practice
```

Якщо у вас немає `kubedojo-practice`, спробуйте:

```bash
$ tree ~ -L 1
```

Прапорець `-L 1` означає «показати лише 1 рівень вглиб» — корисно для великих директорій.

6. **Перевірте, що встановлено:**

На Ubuntu/Debian Linux:
```bash
$ apt list --installed | head -20
```

На macOS:
```bash
$ brew list
```

7. **Пошукайте пакет:**

На Ubuntu/Debian Linux:
```bash
$ apt search "system monitor"
```

На macOS:
```bash
$ brew search "monitor"
```

8. **Перевірте версію встановленого інструмента:**

```bash
$ htop --version
```

Більшість програм підтримують `--version` або `-v` для показу номера версії. Це корисно при діагностиці: «Яку версію цього інструмента я маю?»

### Критерії успіху

Ви виконали цю вправу, коли можете:

- [ ] Оновити менеджер пакетів
- [ ] Встановити `htop` і запустити його (та вийти через `q`)
- [ ] Встановити `tree` і використати для відображення директорії
- [ ] Шукати пакети за ключовим словом
- [ ] Перевірити версію встановленого інструмента

---

> Ви щойно використали інструмент, яким досвідчені інженери користуються щодня. Ви тут на своєму місці.

## Що далі?

Тепер ви знаєте, як програмне забезпечення перетворюється з коду на працюючу програму, як встановлювати інструменти за допомогою менеджера пакетів і що робить `sudo`. Ваш арсенал терміналу зростає.

Відтепер у вас є фундамент для початку вивчення контейнерів, хмарних обчислень і, зрештою, Kubernetes. Кожен інструмент в екосистемі Kubernetes — `kubectl`, `helm`, `kind`, `docker` — встановлюється саме так, як ви щойно навчились.
