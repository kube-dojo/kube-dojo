## Чому цей модуль важливий

У минулому модулі ви створювали файли через `touch` — але вони були порожні. Порожні файли — це як чисті бланки замовлень на кухні ресторану. Корисні, щоб зарезервувати місце, але не більше, поки хтось на них не напише.

У реальному світі вам потрібно буде редагувати файли постійно:
- **Файли конфігурації** повідомляють програмам, як поводитись (як правила роботи кухні)
- **Скрипти** — це файли, що містять послідовність команд (як картка рецепту)
- **Маніфести Kubernetes** — файли, які вказують Kubernetes, що запускати (до цього ви дійдете!)

Вам потрібно вміти відкрити файл, написати в ньому, зберегти й закрити — все з терміналу. Без миші. Без графічного текстового редактора. Лише ви й клавіатура.

## Навіщо редагувати файли в терміналі?

«А хіба не можна просто використати Notepad чи TextEdit?»

Можна — на своєму комп'ютері. Але пам'ятайте з Модуля 0.1: майже кожен сервер працює на Linux, і сервери зазвичай не мають графічного інтерфейсу. Коли ви підключаєтесь до віддаленого сервера (що ви навчитесь у Модулі 0.7), там немає миші, немає робочого столу, немає Notepad. Є лише термінал.

Текстовий редактор у терміналі — це ніж шеф-кухаря в комп'ютерній справі. Не найвишуканіший інструмент, але ви будете використовувати його скрізь.

## Знайомтесь із nano: Ваш перший редактор у терміналі

Існує багато текстових редакторів, що працюють у терміналі. Два найвідоміші — **vim** та **nano**.

Ми починаємо з **nano**, бо:

| nano | vim |
|------|-----|
| Працює так, як ви очікуєте | Має круту криву навчання |
| Друкуєте — і воно друкується | Потрібно натиснути `i`, перш ніж друкувати |
| Меню внизу показує, як зберегти й вийти | Люди відомо застрягають і не можуть зрозуміти, як вийти |
| Ідеально для початківців | Потужний, але спершу приголомшує |

Немає нічого соромного у використанні nano. Багато досвідчених інженерів використовують його для швидких правок. Ви вивчите vim пізніше, коли будете готові до більшої потужності.

> **Цікавий факт**: «Як вийти з vim» — одне з найбільш шуканих програмістських запитань в інтернеті. Понад 2 мільйони людей переглянули це питання на Stack Overflow. З nano такої проблеми у вас не буде.

## Відкриваємо nano

Створімо та відредагуймо файл. Спочатку переконайтесь, що ви в домашній директорії:

```bash
cd ~
```

Тепер відкрийте nano з новим файлом:

```bash
nano hello.txt
```

Ваш екран повністю зміниться. Ви побачите щось на кшталт:

```
  GNU nano 6.2                    hello.txt




^G Help    ^O Write Out  ^W Where Is   ^K Cut       ^C Location
^X Exit    ^R Read File  ^\ Replace    ^U Paste     ^T Execute
```

Розберімо:

- **Верхній рядок**: Назва редактора та назва вашого файлу
- **Середня область**: Тут ви друкуєте (вона порожня, бо файл новий)
- **Два нижні рядки**: Меню з доступними командами

---

## Символ ^ означає Ctrl

Ось єдине, що спершу бентежить усіх:

> **Символ `^` означає «утримуйте клавішу Ctrl».**

Тож коли ви бачите `^O Write Out`, це означає: «Натисніть Ctrl і O одночасно, щоб зберегти файл.»

| Що ви бачите | Що натискаєте | Що це робить |
|-------------|--------------|-------------|
| `^O` | Ctrl + O | Зберегти файл |
| `^X` | Ctrl + X | Вийти з nano |
| `^K` | Ctrl + K | Вирізати поточний рядок |
| `^U` | Ctrl + U | Вставити вирізаний рядок |
| `^W` | Ctrl + W | Пошук тексту |
| `^G` | Ctrl + G | Показати довідку |

Ось і все. Шість комбінацій клавіш — і ви можете зробити все необхідне.

## Введення тексту

Це найпростіша частина: **просто друкуйте.**

З відкритим nano введіть наступне (натискайте Enter наприкінці кожного рядка):

```
Welcome to the Kitchen!
Today's special: Learning to edit files.
Chef says: You're doing great.
```

Ваш екран тепер має показувати:

```
  GNU nano 6.2                    hello.txt                    Modified

Welcome to the Kitchen!
Today's special: Learning to edit files.
Chef says: You're doing great.


^G Help    ^O Write Out  ^W Where Is   ^K Cut       ^C Location
^X Exit    ^R Read File  ^\ Replace    ^U Paste     ^T Execute
```

Зверніть увагу на слово **Modified** у верхньому рядку. Це означає, що ви внесли зміни, які ще не збережені.

---

## Збереження: Ctrl + O

Збережімо. Натисніть:

```
Ctrl + O
```

nano попросить підтвердити назву файлу:

```
File Name to Write: hello.txt
```

Натисніть **Enter** для підтвердження. Індикатор «Modified» зникає. Ваш файл збережено.

---

## Вихід: Ctrl + X

Щоб вийти з nano й повернутися до терміналу:

```
Ctrl + X
```

Якщо ви внесли зміни після останнього збереження, nano запитає:

```
Save modified buffer?  Y Yes  N No  ^C Cancel
```

Натисніть `Y`, щоб зберегти й вийти, `N`, щоб вийти без збереження, або `Ctrl + C`, щоб скасувати й залишитися в nano.

---

## Перевірте файл: `cat`

Тепер, коли ви повернулися в термінал, переконаймося, що файл збережено. Команда `cat` виводить вміст файлу на екран:

```bash
cat hello.txt
```

Очікуваний результат:

```
Welcome to the Kitchen!
Today's special: Learning to edit files.
Chef says: You're doing great.
```

`cat` — скорочення від «concatenate» (об'єднати, бо вона може з'єднувати кілька файлів), але більшість людей використовують її, щоб швидко зазирнути у вміст файлу. Уявіть це як прочитати нотатку, не піднімаючи її — просто кинути на неї погляд.

## Редагування існуючого файлу

Щоб відредагувати файл, який вже існує, просто відкрийте його так само:

```bash
nano hello.txt
```

Ваш текст на місці. Використовуйте стрілки для переміщення. Друкуйте, щоб додати текст. Використовуйте Backspace/Delete для видалення тексту.

Додайте новий рядок унизу:

```
PS: The pantry is fully stocked.
```

Збережіть через `Ctrl + O`, Enter. Вийдіть через `Ctrl + X`.

## Вирізання та вставлення рядків

nano має просте вирізання й вставлення для цілих рядків:

1. Перемістіть курсор на рядок, який хочете вирізати
2. Натисніть **Ctrl + K**, щоб вирізати його (рядок зникає)
3. Перемістіть курсор туди, куди хочете вставити
4. Натисніть **Ctrl + U**, щоб вставити

Можна вирізати кілька рядків, натискаючи `Ctrl + K` кілька разів — вони накопичуються. Потім `Ctrl + U` вставляє їх усі.

Це як шеф-кухар, що змінює порядок кроків у рецепті.

---

## Пошук тексту

Працюєте з довгим файлом і потрібно щось знайти? Натисніть **Ctrl + W**, введіть те, що шукаєте, і натисніть Enter.

```
Ctrl + W
```

nano запитає:

```
Search:
```

Введіть `special` і натисніть Enter. Курсор переміститься до першого входження «special» у файлі.

Натисніть `Ctrl + W` і потім Enter знову (нічого не вводячи), щоб знайти наступне входження.

## Ваш перший скрипт

Тепер зробімо щось потужне: напишемо скрипт. Скрипт — це просто текстовий файл, що містить команди, які комп'ютер може виконати. Це картка рецепту для терміналу.

### Крок 1: Створіть скрипт

```bash
nano my-first-script.sh
```

Введіть наступне точно:

```bash
#!/bin/bash

echo "Welcome to the kitchen!"
echo "Today's date is: $(date)"
echo "You are logged in as: $(whoami)"
echo "Your current directory is: $(pwd)"
echo ""
echo "Great job, chef! Your first script works!"
```

Розберімо кожен рядок:

- `#!/bin/bash` — Це називається «shebang». Він повідомляє комп'ютеру: «Використовуй програму bash для виконання цього файлу.» Кожен скрипт потребує цього як першого рядка. (Так, «shebang» — це справжній термін.)
- `echo` — Команда, що виводить текст на екран. Уявіть це як кухню, що вигукує «Замовлення готове!»
- `$(date)` — Виконує команду `date` і підставляє результат. Синтаксис `$()` означає «виконай цю команду й дай мені результат».
- `$(whoami)` — Виконує `whoami`, що показує ваше ім'я користувача.
- `$(pwd)` — Ви це знаєте — виводить вашу поточну директорію.

### Крок 2: Збережіть і вийдіть

Натисніть `Ctrl + O`, Enter, потім `Ctrl + X`.

### Крок 3: Зробіть його виконуваним

Зараз файл — це просто текст. Комп'ютер не запустить його, бо у нього немає *дозволу* на виконання. Виправимо це:

```bash
chmod +x my-first-script.sh
```

`chmod` означає **ch**ange **mod**e (змінити режим). `+x` означає «додати дозвіл на виконання». Ви кажете менеджеру ресторану: «Це не просто документ — це рецепт, який потрібно приготувати.»

### Крок 4: Запустіть його

```bash
./my-first-script.sh
```

`./` означає «запустити цей файл із поточної директорії». Очікуваний результат:

```
Welcome to the kitchen!
Today's date is: Sun Mar 23 14:30:00 UTC 2026
You are logged in as: yourname
Your current directory is: /Users/yourname

Great job, chef! Your first script works!
```

Ви щойно написали й виконали свою першу програму. Це не дрібниця — це все.

## Примітка про vim

Рано чи пізно ви зіткнетесь з vim. Він неймовірно потужний, використовується багатьма досвідченими інженерами і встановлений практично на кожній системі Linux. Але vim має відомо круту криву навчання, бо працює в «режимах» — не можна просто відкрити й почати друкувати.

Ми не ігноруємо vim. Ми залишаємо його на потім, коли у вас буде більше контексту. Зараз nano справляється, і вчитися ходити перед бігом — так тренуються і шеф-кухарі.

Коли ви будете готові, vim буде на місці. Наразі nano — ваш друг.

## Чи знали ви?

- **nano — це клон клону.** Його створили як безплатну заміну редактору `pico`, який був вбудований у поштовий клієнт Pine з 1990-х. `nano` означає «nano's ANOther editor» (ще один редактор nano) — рекурсивний акронім, що є гіківською традицією у програмуванні.

- **Найстаріший текстовий редактор, що досі використовується — `ed`, створений у 1969 році.** Його написав Кен Томпсон, один зі творців Unix. `ed` — це «рядковий редактор»: ви не можете бачити весь файл одразу. Ви редагуєте один рядок за раз. Інженери у 1970-х писали цілі операційні системи за допомогою `ed`. Ваш досвід із nano — розкіш у порівнянні.

- **Файли конфігурації керують світом.** Майже кожна програма зчитує текстовий файл конфігурації при запуску. Ваш вебсервер, ваша база даних, сам Kubernetes — усі вони читають текстові файли, щоб знати, як поводитись. Уміння редагувати ці файли з терміналу — одна з найпрактичніших навичок у всій комп'ютерній справі.

## Типові помилки

| Помилка | Чому це проблема | Що робити натомість |
|---------|-----------------|---------------------|
| Натискаєте Ctrl+Z замість Ctrl+X для виходу | Ctrl+Z призупиняє nano (ховає його) замість закриття. Файл залишається відкритим у фоні | Якщо випадково призупинили, введіть `fg`, щоб повернути nano. Потім використайте Ctrl+X для правильного виходу |
| Забуваєте зберегти перед виходом | Ваші зміни втрачені | Завжди Ctrl+O для збереження перед Ctrl+X для виходу. Або просто натисніть Ctrl+X і скажіть Y, коли запитають про збереження |
| Не додаєте `#!/bin/bash` до скриптів | Система не знає, як запустити файл | Завжди робіть першим рядком bash-скрипту `#!/bin/bash` |
| Забуваєте `chmod +x` перед запуском скрипту | Отримуєте «Permission denied» | Виконайте `chmod +x filename.sh`, щоб зробити файл виконуваним |
| Використовуєте `nano`, коли хотіли `cat` | Випадково відкриваєте файл для редагування, коли просто хотіли його прочитати | Використовуйте `cat filename` для перегляду файлу без редагування |

## Тест

1. **Що означає символ `^` у меню nano?**
   <details>
   <summary>Відповідь</summary>
   Символ `^` означає клавішу Ctrl. Тож `^O` означає «натисніть Ctrl і O одночасно», а `^X` означає «натисніть Ctrl і X одночасно». Це спосіб nano показувати клавіатурні скорочення.
   </details>

2. **Як зберегти файл у nano?**
   <details>
   <summary>Відповідь</summary>
   Натисніть Ctrl+O (показується як ^O у меню nano), потім натисніть Enter для підтвердження назви файлу. Це записує файл на диск. Індикатор «Modified» у верхньому рядку зникне після збереження.
   </details>

3. **Що таке «shebang» і навіщо він потрібен у скриптах?**
   <details>
   <summary>Відповідь</summary>
   Shebang — це рядок `#!/bin/bash` на самому початку скрипту. Він повідомляє операційній системі, яка програма має інтерпретувати й виконувати скрипт. Без нього система не знає, чи це bash-скрипт, Python-скрипт чи щось інше. Назва походить від символів «#!»: hash + bang = shebang.
   </details>

4. **Що робить `chmod +x` і навіщо це потрібно?**
   <details>
   <summary>Відповідь</summary>
   `chmod +x` додає дозвіл на виконання файлу. За замовчуванням нові текстові файли — це просто дані: система не запустить їх як програми. Додавання дозволу на виконання каже системі: «Цей файл містить інструкції, які потрібно виконати.» Без цього ви отримаєте помилку «Permission denied» при спробі запустити скрипт.
   </details>

5. **Яка різниця між `nano hello.txt` та `cat hello.txt`?**
   <details>
   <summary>Відповідь</summary>
   `nano hello.txt` відкриває файл у інтерактивному редакторі, де можна змінювати вміст, зберігати й виходити. `cat hello.txt` просто виводить вміст файлу на екран терміналу й одразу повертає до рядка запрошення — це лише перегляд, без можливості редагування. Використовуйте `cat`, коли хочете подивитися, і `nano`, коли хочете змінити.
   </details>

## Практична вправа: Дошка оголошень кухні

Створіть «дошку оголошень» для кухні ресторану, попрактикуйте редагування та напишіть скрипт.

### Частина 1: Створіть і відредагуйте оголошення

```bash
cd ~
nano kitchen-memo.txt
```

Введіть наступні п'ять рядків:

```
=== KITCHEN MEMO BOARD ===
1. Morning prep starts at 6 AM
2. New menu items arriving Thursday
3. Remember: clean as you go
4. Staff meeting at 3 PM Friday
```

Збережіть через `Ctrl + O`, Enter. Вийдіть через `Ctrl + X`.

Перевірте свою роботу:

```bash
cat kitchen-memo.txt
```

Ви маєте побачити всі п'ять рядків, надрукованих саме так, як ви їх ввели.

### Частина 2: Відредагуйте оголошення

Відкрийте його знову:

```bash
nano kitchen-memo.txt
```

Додайте шостий рядок унизу:

```
5. Chef says: great work today, team!
```

Збережіть і вийдіть (`Ctrl + O`, Enter, `Ctrl + X`).

Перевірте:

```bash
cat kitchen-memo.txt
```

Мають бути всі шість рядків.

### Частина 3: Напишіть скрипт для звіту

```bash
nano kitchen-report.sh
```

Введіть:

```bash
#!/bin/bash

echo "=== Kitchen Status Report ==="
echo "Date: $(date)"
echo "Chef on duty: $(whoami)"
echo ""
echo "--- Memo Board Contents ---"
cat kitchen-memo.txt
echo ""
echo "--- Files in current directory ---"
ls
echo ""
echo "Report complete. Kitchen is running smoothly!"
```

Збережіть і вийдіть.

Зробіть виконуваним і запустіть:

```bash
chmod +x kitchen-report.sh
./kitchen-report.sh
```

Очікуваний результат (деталі відрізнятимуться):

```
=== Kitchen Status Report ===
Date: Sun Mar 23 14:45:00 UTC 2026
Chef on duty: yourname
```
