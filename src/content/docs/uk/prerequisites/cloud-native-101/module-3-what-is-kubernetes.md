## Чому цей модуль важливий

Ви знаєте, що таке контейнери. Ви вмієте збирати та запускати їх за допомогою Docker. Але Docker запускає контейнери на ОДНІЙ машині. Що робити, коли потрібно:
- Сотні контейнерів?
- Висока доступність (без простоїв)?
- Автоматичне масштабування?
- Кілька машин?

Ось тут і з'являється Kubernetes. Цей модуль дає загальну картину перед зануренням у деталі.

---

## Проблема: контейнери в масштабі

Docker чудово підходить для запуску кількох контейнерів на вашому ноутбуці. Але продакшен потребує більшого:

```
┌─────────────────────────────────────────────────────────────┐
│              SINGLE MACHINE LIMITATIONS                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Your Docker Host:                                         │
│  ┌────────────────────────────────────────────────────┐    │
│  │  🐳 Container 1  🐳 Container 2  🐳 Container 3    │    │
│  │  🐳 Container 4  🐳 Container 5  🐳 Container 6    │    │
│  └────────────────────────────────────────────────────┘    │
│                                                             │
│  Problems:                                                  │
│  ❌ Machine dies = ALL containers die                      │
│  ❌ Need more capacity? Buy bigger machine                 │
│  ❌ No automatic recovery                                  │
│  ❌ Manual load balancing                                  │
│  ❌ Updates require downtime                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Що потрібно продакшену

```
✅ Run containers across multiple machines
✅ Automatic restart when containers crash
✅ Automatic placement (which machine has capacity?)
✅ Load balancing between containers
✅ Rolling updates without downtime
✅ Scaling up/down based on demand
✅ Self-healing when things break
```

Це **оркестрація контейнерів**, і Kubernetes — галузевий стандарт.

## Що таке Kubernetes?

Kubernetes (K8s) — це платформа оркестрації контейнерів із відкритим вихідним кодом. Вона:

1. **Керує розгортанням контейнерів** на кількох машинах
2. **Забезпечує бажаний стан** — якщо щось ламається, K8s це виправляє
3. **Забезпечує мережу** — контейнери знаходять одне одного й спілкуються
4. **Керує сховищем** — постійні дані для застосунків зі станом <!-- VERIFY: застосунків зі станом (= stateful apps) -->
5. **Автоматизує операції** — масштабування, оновлення, відновлення

### Аналогія: робота аеропорту

```
┌─────────────────────────────────────────────────────────────┐
│              KUBERNETES AS AIRPORT OPERATIONS               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Airport Control Tower = Kubernetes Control Plane          │
│  - Assigns planes to gates (schedules pods to nodes)        │
│  - Monitors all activity (tracks cluster state)             │
│  - Responds to incidents (restarts failed containers)       │
│                                                             │
│  Gates/Runways = Worker Nodes                               │
│  - Physical infrastructure where work happens               │
│  - Control tower assigns, gates execute                     │
│                                                             │
│  Planes = Pods (containers)                                 │
│  - Arrive, depart, get serviced                            │
│  - Control tower tracks status of each                      │
│                                                             │
│  Airlines = Namespaces                                      │
│  - Delta uses gates 1-10                                   │
│  - United uses gates 11-20                                 │
│  - Isolation between tenants                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Компоненти площини керування

| Компонент | Що він робить |
|-----------|---------------|
| **API Server** | Вхідні двері до K8s. Усі команди проходять через нього. |
| **etcd** | База даних, що зберігає весь стан кластера |
| **Scheduler** | Вирішує, яка нода запускає кожний под |
| **Controller Manager** | Забезпечує відповідність бажаного стану фактичному |

### Компоненти робочої ноди

| Компонент | Що він робить |
|-----------|---------------|
| **kubelet** | Агент на кожній ноді, керує подами |
| **Container Runtime** | Безпосередньо запускає контейнери (containerd) |
| **kube-proxy** | Забезпечує мережу для сервісів |

## Попередній огляд основних концепцій

### Поди
Найменша одиниця розгортання. Зазвичай один контейнер, іноді — кілька пов'язаних контейнерів.

```yaml
# You don't run containers directly—you create Pods
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  containers:
  - name: nginx
    image: nginx:1.25
```

### Деплойменти
Керують кількома ідентичними подами. Забезпечують оновлення та масштабування.

```yaml
# "I want 3 nginx pods, always"
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.25
```

### Сервіси
Стабільна мережа для подів. Поди з'являються і зникають; Сервіси забезпечують постійний доступ.

```yaml
# "Make my nginx pods accessible on port 80"
apiVersion: v1
kind: Service
metadata:
  name: nginx
spec:
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
```

## Чому не просто Docker?

| Функція | Docker (один хост) | Kubernetes |
|---------|---------------------|------------|
| Кілька нод | --- | Так |
| Автомасштабування | --- | Так |
| Самовідновлення | --- | Так |
| Плавні оновлення | Вручну | Автоматично |
| Балансування навантаження | Вручну | Вбудоване |
| Виявлення сервісів | Вручну | Вбудоване |
| Керування секретами | --- | Так |
| Обмеження ресурсів | На контейнер | На весь кластер |

Docker призначений для запуску контейнерів. Kubernetes — для експлуатації контейнерів у масштабі.

## Де працює Kubernetes

### Хмарні керовані сервіси (найпростіше)
- **AWS**: EKS (Elastic Kubernetes Service)
- **Google Cloud**: GKE (Google Kubernetes Engine)
- **Azure**: AKS (Azure Kubernetes Service)

Хмарні провайдери керують площиною керування. Ви просто запускаєте робочі навантаження.

### Самостійне керування
- **kubeadm**: Офіційний інсталятор K8s
- **k3s**: Легкий K8s (підходить для edge/IoT)
- **OpenShift**: Корпоративний K8s від Red Hat

Ви керуєте всім самостійно.

### Локальна розробка
- **kind**: Kubernetes у Docker
- **minikube**: Локальний K8s у ВМ/контейнері
- **Docker Desktop**: Містить опцію K8s

Для навчання та розробки.

## Чи знали ви?

- **«Kubernetes» — грецьке слово, що означає «керманич»** (той, хто керує кораблем). Логотип — це штурвал із 7 спицями — за кількістю початкових розробників.

- **K8s — це нумеронім.** K-[8 літер]-s. Як i18n (internationalization) або a11y (accessibility).

- **Google запускає все на Kubernetes.** Gmail, Search, YouTube — усе на Borg (попередник K8s) або K8s.

- **Найбільший відомий кластер K8s** має 15 000 нод, на яких працюють понад 300 000 подів (Alibaba).

## Поширені хибні уявлення

| Хибне уявлення | Реальність |
|----------------|------------|
| «K8s замінює Docker» | K8s оркеструє контейнери. Ви й далі використовуєте Docker для збірки образів. |
| «K8s тільки для великих компаній» | Малі стартапи теж використовують K8s. Керовані сервіси роблять це доступним. |
| «K8s — це складно» | K8s ДІЙСНО складний, але керовані сервіси беруть на себе більшість складності. |
| «K8s розв'язує все» | K8s — це інфраструктура. Вам усе одно потрібно проєктувати якісні застосунки. |

## Тест

1. **Яку проблему Kubernetes розв'язує, а Docker сам — ні?**
   <details>
   <summary>Відповідь</summary>
   Запуск контейнерів на кількох машинах з автоматичним плануванням, самовідновленням, масштабуванням, балансуванням навантаження та оновленнями без простою. Docker сам по собі керує контейнерами лише на одному хості.
   </details>

2. **Який зв'язок між Подом і контейнером?**
   <details>
   <summary>Відповідь</summary>
   Под — це найменша одиниця розгортання в Kubernetes, яка містить один або кілька контейнерів. Ви не розгортаєте контейнери безпосередньо — ви створюєте Поди. Контейнери в Поді мають спільну мережу та сховище.
   </details>

3. **Що робить Scheduler у Kubernetes?**
   <details>
   <summary>Відповідь</summary>
   Scheduler вирішує, яка робоча нода має запустити новий под. Він враховує доступність ресурсів, обмеження, правила спорідненості та інші чинники для оптимального розміщення.
   </details>

4. **Навіщо використовувати керований сервіс Kubernetes (EKS, GKE, AKS)?**
   <details>
   <summary>Відповідь</summary>
   Керовані сервіси забезпечують роботу площини керування (API server, etcd, scheduler, controllers). Вам не потрібно турбуватися про доступність площини керування, оновлення чи обслуговування. Ви просто запускаєте робочі навантаження.
   </details>

## Практична вправа

**Завдання**: Дослідити кластер Kubernetes (попередній огляд того, що буде далі).

```bash
# If you have a cluster running (kind, minikube, or other):

# 1. See your cluster nodes
kubectl get nodes
# Output shows the machines in your cluster

# 2. See running system components
kubectl get pods -n kube-system
# These are the components that make K8s work

# 3. See all namespaces (like folders for resources)
kubectl get namespaces

# 4. Create something simple
kubectl run hello --image=nginx --restart=Never
kubectl get pods
# You just created a Pod!

# 5. See what Kubernetes knows about it
kubectl describe pod hello
# Lots of information about scheduling, containers, events

# 6. Clean up
kubectl delete pod hello

# Don't worry if this is confusing now - you'll learn all of it
# The goal is just to see K8s in action
```

**Ще немає кластера?** Нічого! Трек «Основи Kubernetes» проведе вас через його налаштування. Це лише попередній огляд.

**Критерій успіху**: Побачити, що Kubernetes надає API для керування контейнерами на кількох машинах.

## Підсумок

Kubernetes — це платформа оркестрації контейнерів, яка:

- **Планує** розподіл контейнерів по кількох машинах
- **Самовідновлюється**, перезапускаючи контейнери, що впали
- **Масштабується** відповідно до попиту
- **Балансує навантаження** трафіку до контейнерів
- **Оновлює** без простою
- **Керує** мережею та сховищем

Ключові концепції:
- **Кластер**: площина керування + робочі ноди
- **Под**: найменша одиниця розгортання
- **Деплоймент**: керує реплікованими подами
- **Сервіс**: стабільна мережа для подів
