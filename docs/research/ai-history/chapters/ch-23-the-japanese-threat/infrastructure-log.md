# Infrastructure Log: Chapter 23 - The Japanese Threat

## Technical Stack

- **Knowledge information processing:** The application layer FGCS was trying to
  serve. It included AI-adjacent tasks, knowledge bases, natural language, legal
  reasoning, theorem proving, and advanced problem solving.
- **Logic programming bridge:** Furukawa and Kurozumi frame logic programming as
  the interface between knowledge information processing and highly parallel
  architecture. This is the central infrastructure idea.
- **Kernel language:** Fuchi's term for the machine-language equivalent of a new
  predicate-logic-based architecture. It was not meant to be the end-user
  programming language.
- **KLO / ESP:** Initial-stage sequential logic-programming infrastructure. PSI
  was a KLO machine; ESP was a more user-friendly language; SIMPOS was written
  in ESP.
- **GHC / KL1:** GHC was a concurrent/parallel logic language model; KL1 became
  the language integrating the final-stage research plan.
- **PSI / SIMPOS:** Personal Sequential Inference Machine and its operating
  system. Use PSI as the workstation/research platform and SIMPOS as the example
  of an operating system written in a logic-programming language.
- **Multi-PSI:** Intermediate infrastructure for software research and
  demonstrations.
- **PIM / PIMOS:** Parallel Inference Machine prototypes and operating system in
  the final stage. PIMOS and KL1 are the mature technical center of the project.
- **Kappa / Quixote / MGTP / HELIC-II:** Application and knowledge-processing
  systems that can be used selectively to show real technical output, especially
  theorem proving and legal reasoning.
- **ICOT Free Software / KLIC:** Follow-on dissemination path. KLIC compiled KL1
  to C for Unix-based machines, making FGCS software less dependent on PIM
  hardware.

## Infrastructure Lesson

FGCS tried to make symbolic AI into a full computing stack: language, operating
system, processor architecture, knowledge base, development environment, and
applications all aligned around logic programming and parallel inference. Its
historical importance lies in that stack-level ambition. Its limitation was that
the stack was hard to turn into broad commercial infrastructure before ordinary
hardware, Unix environments, and other programming models kept moving.
