### Architecture consult (codex gpt-5.5, full-auto, kubedojo cwd)

Channel `878-arch-consult`. Verdict: "Plan is sound with refinements." Key refinements applied:

- **Set-based diff, not count-based.** Pure count check is too weak — it misses the case where one old `(kind, incident, file)` triple is removed and a new one is added (count nets zero, but a NEW violation entered). - **Per-file directives, not free-pick replacements.** Brief specifies for each duplicate: incident name + canonical owner + chosen action (b/c). Codex doesn't pick from the catalog autonomously.
- **Don't do one giant content PR.** Codex flagged the repo's <20-file convention. We split sweep into Phases 1/2/3 by risk/action, not by track.
- **Cross-ref marker proximity verified.** Codex actually inspected `scripts/check_incident_reuse.py` and confirmed the marker is recognized within 200 chars of the duplicate match (only for incidents already in CANONICALS).
- **Forbidden-trope coverage.** Set-based delta naturally covers `kind: forbidden` since the key includes `kind`.
- **Threat model gap.** Catalog replacements need scanner-pattern coverage in `audit_incident_reuse.py` AND a CANONICALS claim, otherwise they silently re-duplicate later. Phase 2 didn't need this because all duplicates collapsed to existing canonicals via action (b)/(c) — but it's the rule for any future xref→a transition.
