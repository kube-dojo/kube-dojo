## Запитання

### Налаштування кластера (Модуль 0.1)

1. **Яка команда ініціалізує вузол площини управління Kubernetes?**
   <details>
   <summary>Відповідь</summary>
   `kubeadm init` — опціонально з прапорцями, як-от `--pod-network-cidr` для CNI
   </details>

2. **Де зберігаються маніфести статичних Подів на вузлі площини управління?**
   <details>
   <summary>Відповідь</summary>
   `/etc/kubernetes/manifests/`
   </details>

3. **Що станеться, якщо видалити kube-scheduler.yaml з каталогу маніфестів?**
   <details>
   <summary>Відповідь</summary>
   Под планувальника зупиниться, і нові Поди залишаться у стані Pending, оскільки ніщо не розподілятиме їх по вузлах.
   </details>

4. **Який CNI-плагін kind використовує за замовчуванням?**
   <details>
   <summary>Відповідь</summary>
   kindnet (простий CNI для локальної розробки)
   </details>

### Майстерність у Shell (Модуль 0.2)

5. **Який стандартний аліас kubectl використовується при підготовці до CKA?**
   <details>
   <summary>Відповідь</summary>
   `alias k=kubectl`
   </details>

6. **У що зазвичай розгортається змінна `$do`?**
   <details>
   <summary>Відповідь</summary>
   `--dry-run=client -o yaml` — використовується для генерації YAML без створення ресурсів
   </details>

7. **Як увімкнути автодоповнення kubectl для аліасу `k`?**
   <details>
   <summary>Відповідь</summary>
   `complete -o default -F __start_kubectl k`
   </details>

8. **Яке скорочене ім'я для persistentvolumeclaim?**
   <details>
   <summary>Відповідь</summary>
   `pvc`
   </details>

### Vim для YAML (Модуль 0.3)

9. **Яке налаштування vim забезпечує перетворення табуляцій на пробіли?**
   <details>
   <summary>Відповідь</summary>
   `set expandtab`
   </details>

10. **Ви вставили YAML і відступи зламалися. Яку команду vim слід виконати ПЕРЕД наступною вставкою?**
    <details>
    <summary>Відповідь</summary>
    `:set paste` (і `:set nopaste` після вставки)
    </details>

11. **Яка команда vim видаляє 5 рядків від позиції курсору?**
    <details>
    <summary>Відповідь</summary>
    `5dd`
    </details>

12. **Яка команда vim зберігає файл і виходить?**
    <details>
    <summary>Відповідь</summary>
    `:wq` (або `ZZ` у звичайному режимі)
    </details>

### Навігація документацією (Модуль 0.4)

13. **Який розділ kubernetes.io містить покрокові інструкції?**
    <details>
    <summary>Відповідь</summary>
    Tasks (`kubernetes.io/docs/tasks/`)
    </details>

14. **Яка команда kubectl показує доступні поля ресурсу без інтернету?**
    <details>
    <summary>Відповідь</summary>
    `kubectl explain <resource>` (наприклад, `kubectl explain pod.spec`)
    </details>

15. **Де знайти документацію Helm під час іспиту?**
    <details>
    <summary>Відповідь</summary>
    `helm.sh/docs/`
    </details>

16. **Чи доступна документація Gateway API під час іспиту CKA?**
    <details>
    <summary>Відповідь</summary>
    Так, вона є частиною документації kubernetes.io у розділі Concepts → Services → Gateway API
    </details>

### Стратегія іспиту (Модуль 0.5)

17. **Які три проходи у методі трьох проходів?**
    <details>
    <summary>Відповідь</summary>
    1. Швидкі перемоги (завдання на 1–3 хв)
    2. Середні завдання (завдання на 4–6 хв)
    3. Складні завдання (завдання на 8–15 хв)
    </details>

18. **Що ПЕРШЕ слід зробити, починаючи БУДЬ-ЯКЕ завдання іспиту?**
    <details>
    <summary>Відповідь</summary>
    Переключитися на правильний контекст кластера: `kubectl config use-context <context-name>`
    </details>

19. **Який прохідний бал CKA?**
    <details>
    <summary>Відповідь</summary>
    66%
    </details>

20. **У завданні сказано: «Виявіть, чому Поди не запускаються.» До якого проходу це належить?**
    <details>
    <summary>Відповідь</summary>
    Прохід 3 (Складний) — завдання з усунення неполадок потребують дослідження і непередбачувані за часом
    </details>
