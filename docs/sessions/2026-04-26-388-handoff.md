## What NOT to do (lessons learned)

- **Don't run two parallel batches on the same v2 queue.** The merge_lock + ledger-row-commit window isn't airtight; produces compounding races. Single batch with workers=1 is stable.
- **Don't reset state by clearing individual queue keys.** `pop('assigned_writer')` while leaving the `queue` dict intact bricks `ensure_queued`'s idempotency check (it sees `queue` key present, doesn't re-init, then `claim()` KeyErrors). Pop the entire `queue` key — `ensure_queued` rebuilds.
- **Don't trust the visual-aid gate's per-type strictness for batch operation.** Codex deterministically converts ASCII to tables on certain content; the gate now allows total-preserved substitutions, and the prompt forbids substitution explicitly. If both fail for a module, manual rewrite (or Mermaid pre-strip) is the path — not a third re-attempt.
- **Don't run `review` against Anthropic when 30+ consecutive `rc=1`s in a row.** That's a quota signal even when the error message is empty. Switch to `KUBEDOJO_SKIP_REVIEW=1` and defer.
