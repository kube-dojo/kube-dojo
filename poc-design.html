Module 3.4: Ingress

⚠ Medium ⏱ 45-55 min 🏆 CKA Exam

Why This Module Matters

While Services expose applications to the network, Ingress provides HTTP/HTTPS routing with features like path-based routing, virtual hosts, and TLS termination. Instead of exposing many NodePorts, you expose one Ingress controller that routes traffic based on URLs.

⚠️ Important
The ingress-nginx controller reached end-of-life on March 31, 2026. The Ingress API itself is NOT deprecated — it's part of the Kubernetes stable API. Consider migrating to the Gateway API for new projects.
💡 Did You Know?
The Gateway API took 4 years to reach GA (v1.0 in October 2023). It was originally called "Service APIs" and renamed to avoid confusion with the existing Service resource. The design required consensus from 8 different Ingress controller vendors.

Part 1: Ingress Architecture

1.1 How Ingress Works

An Ingress resource defines routing rules. An Ingress Controller watches for these resources and configures an actual load balancer (nginx, envoy, HAProxy) accordingly.

yaml
apiVersion: networking.k8s.io/v1 kind: Ingress metadata: name: web-ingress annotations: nginx.ingress.kubernetes.io/rewrite-target: / spec: ingressClassName: nginx rules: - host: "app.example.com" http: paths: - path: /api pathType: Prefix backend: service: name: api-service port: number: 8080
🔥 War Story: The Ingress That Brought Down Black Friday $2.3M lost revenue

A major e-commerce platform used a single nginx Ingress controller for all traffic. On Black Friday 2023, a misconfigured rate-limiting annotation on one backend caused the entire controller to reload every 3 seconds, dropping connections across all 47 services behind it.

09:00
Traffic ramp begins
11:23
Nginx reload loop starts
11:24

11:51

12:05
Full recovery

Lesson: Never share a single Ingress controller across environments. Use separate IngressClasses for critical vs. non-critical traffic.

Hands-On Exercise

Hands-On Deploy an Ingress Controller on kind Exercise 1 of 3
Create a kind cluster with port mappings:
$ kind create cluster --config=- < kind: Cluster apiVersion: kind.x-k8s.io/v1alpha4 nodes: - role: control-plane kubeadmConfigPatches: - | kind: InitConfiguration EOF
Install the nginx Ingress controller:
$ kubectl apply -f https://raw.githubusercontent.com/ kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml
Wait for the controller to be ready:
$ kubectl wait --namespace ingress-nginx \ --for=condition=ready pod \ --selector=app.kubernetes.io/component=controller \ --timeout=90s

YAML Challenge Complete the TLS Ingress Q2 of 8

Fill in the blanks to configure TLS termination for this Ingress resource:

apiVersion: networking.k8s.io/v1 kind: Ingress spec: tls: - hosts: - ? secretName: ? rules: - host: app.example.com http: paths: - path: / pathType: ?
💡 Hint
The host in the TLS section must match the host in the rules. The secretName refers to a Kubernetes Secret of type kubernetes.io/tls.
